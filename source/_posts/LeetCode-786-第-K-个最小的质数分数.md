LeetCode 第 786 题：[第 K 个最小的质数分数](https://leetcode.cn/problems/k-th-smallest-prime-fraction/)。
解法的核心在于**对分数的值进行二分**，结合双指针统计数量，并在统计过程中记录满足条件的最大分数。
{% endcenterquote %}
<!--more-->

### 问题描述

给你一个按递增顺序排序的数组 `arr` 和一个整数 `k` 。数组 `arr` 由 `1` 和若干 **质数** 组成，且其中所有整数互不相同。

对于每对满足 `0 <= i < j < arr.length` 的 `i` 和 `j` ，可以得到分数 `arr[i] / arr[j]` 。

那么第 `k` 个最小的分数是多少呢?  以长度为 2 的整数数组返回你的答案, 这里 `answer[0] == arr[i]` 且 `answer[1] == arr[j]` 。

**示例 1：**
```text
输入：arr = [1,2,3,5], k = 3
输出：[2,5]
解释：已构造好的分数,排序后如下所示: 
1/5, 1/3, 2/5, 1/2, 3/5, 2/3
很明显第三个最小的分数是 2/5
```

**示例 2：**
```text
输入：arr = [1,7], k = 1
输出：[1,7]
```

**提示：**
*   `2 <= arr.length <= 1000`
*   `1 <= arr[i] <= 3 * 10^4`
*   `arr[0] == 1`
*   `arr[i]` 是一个 **质数** ，`i > 0`
*   `arr` 中的所有数字 **互不相同** ，且按 **严格递增** 排序
*   `1 <= k <= arr.length * (arr.length - 1) / 2`

### 解题思路

#### 1. 思路分析
这道题要求找出第 `k` 小的分数。如果我们将所有可能的分数 `arr[i] / arr[j]` 列出来并排序，时间复杂度会非常高，因为分数的总对数是 `O(N^2)` 级别的。

我们可以换一个角度：
分数的范围一定在 `(0, 1)` 之间。我们可以对**数值**进行二分查找。
假设我们选定一个值 `mid`，我们可以很容易地统计出有多少个分数严格小于 `mid`。

*   如果小于 `mid` 的分数个数等于 `k`，那么 `mid` 附近的某个分数可能就是答案吗？不一定，因为 `mid` 是一个浮点数，它可能不等于任何一个 `arr[i] / arr[j]`。
*   但是，如果我们能在统计“小于 `mid` 的分数”的过程中，同时记录下**小于 `mid` 的最大分数**，那么当计数恰好为 `k` 时，这个记录下来的最大分数就是我们要找的第 `k` 小的分数。

#### 2. 二分查找流程
1.  **初始化范围**：`l = 0.0`, `r = 1.0`。
2.  **二分过程**：
    *   取 `mid = (l + r) / 2`。
    *   **统计与记录**：遍历分母 `arr[j]`（从索引 1 到 n-1），使用双指针（或滑动窗口）找到对应的分子 `arr[i]`，使得 `arr[i] / arr[j] < mid`。
    *   在遍历过程中，计算所有满足条件的分数总数 `cnt`。
    *   **关键点**：在统计过程中，维护一个变量 `x/y`，用来记录当前小于 `mid` 的所有分数中**最大**的那个分数。
3.  **调整边界**：
    *   如果 `cnt < k`：说明 `mid` 选小了，甚至还没凑够 `k` 个数，需要扩大范围，`l = mid`。
    *   如果 `cnt > k`：说明 `mid` 选大了，`r = mid`。
    *   如果 `cnt == k`：说明我们要找的第 `k` 小的分数包含在小于 `mid` 的集合中，并且由于我们记录了集合中最大的那个分数 `x/y`，它一定就是第 `k` 小的分数（因为比它小的有 `k-1` 个）。直接返回 `[x, y]`。

#### 3. 双指针的运用
由于数组 `arr` 是严格递增的：
对于固定的分母 `arr[j]`，随着分子索引 `i` 的增加，`arr[i] / arr[j]` 是单调递增的。
因此，对于每个 `j`，我们不需要每次都从头扫描 `i`。
随着 `j` 的增加（分母变大），为了保持 `arr[i] / arr[j] < mid`，`i` 只会向右移动（分子变大）或保持不变。这使得内部统计的时间复杂度为 `O(N)`。

```python
class Solution:
    def kthSmallestPrimeFraction(self, arr: List[int], k: int) -> List[int]:
        n = len(arr)
        # 二分查找的范围初始化为 (0.0, 1.0)
        l, r = 0.0, 1.0
        
        while True:
            mid = (l + r) / 2
            # i 是分子的索引，cnt 统计小于 mid 的分数个数
            i, cnt = -1, 0
            # x, y 用于记录小于 mid 的最大分数，初始设为 0/1
            x, y = 0, 1
            
            # 遍历每一个可能的分母 arr[j]
            for j in range(1, n):
                # 寻找当前分母下，满足分数 < mid 的最大分子索引 i
                # arr 是递增的，所以 i 是单调不减的（双指针逻辑）
                while arr[i+1] / arr[j] < mid:
                    i += 1
                    # 更新当前小于 mid 的最大分数
                    # 比较 arr[i]/arr[j] 与 x/y 的大小
                    # 使用交叉相乘 arr[i]*y > arr[j]*x 避免浮点精度问题
                    if arr[i] * y > arr[j] * x:
                        x, y = arr[i], arr[j]
                
                # arr[0] 到 arr[i] 都是满足条件的分子，共 i+1 个
                cnt += i + 1
            
            # 如果小于 mid 的分数个数恰好为 k
            # 那么我们在统计过程中记录的最大分数 x/y 即为第 k 小的分数
            if cnt == k:
                return [x, y]
            
            # 如果个数不足 k，说明 mid 偏小，需要向右区间搜索
            if cnt < k:
                l = mid
            # 如果个数超过 k，说明 mid 偏大，需要向左区间搜索
            else:
                r = mid
```

<!-- tab Go -->
```go
func kthSmallestPrimeFraction(arr []int, k int) []int {
    n := len(arr)
    // 二分查找的范围初始化为 (0.0, 1.0)
    l, r := 0.0, 1.0

    for {
        mid := (l + r) / 2
        // i 是分子的索引
        // cnt 统计小于 mid 的分数个数
        i, cnt := -1, 0
        // x, y 用于记录小于 mid 的最大分数，初始设为 0/1
        // x 为分子，y 为分母
        x, y := 0, 1

        // 遍历每一个可能的分母 arr[j]
        for j := 1; j < n; j++ {
            // 寻找当前分母下，满足分数 < mid 的最大分子索引 i
            // 需要将整数转换为 float64 进行除法比较
            // 注意边界检查：i+1 < n 且 arr[i+1]/arr[j] < mid
            for i+1 < j && float64(arr[i+1])/float64(arr[j]) < mid {
                i++
                // 更新当前小于 mid 的最大分数
                // 比较 arr[i]/arr[j] 与 x/y 的大小
                // 使用交叉相乘 arr[i]*y > arr[j]*x，利用整数运算避免精度丢失
                if arr[i]*y > arr[j]*x {
                    x, y = arr[i], arr[j]
                }
            }
            // arr[0] 到 arr[i] 都是满足条件的分子，共 i+1 个
            cnt += i + 1
        }

        // 如果小于 mid 的分数个数恰好为 k
        // 那么我们在统计过程中记录的最大分数 x/y 即为第 k 小的分数
        if cnt == k {
            return []int{x, y}
        }

        // 如果个数不足 k，说明 mid 偏小，需要向右区间搜索
        if cnt < k {
            l = mid
        } else {
            // 如果个数超过 k，说明 mid 偏大，需要向左区间搜索
            r = mid
        }
    }
}
```

### 复杂度分析

*   **时间复杂度**: `O(N * log(C))`
    其中 `N` 是数组的长度，`C` 取决于二分的精度要求或者数据的数值范围。由于每次 check 操作需要遍历数组一次（双指针复杂度为 `O(N)`），而二分查找在实数域上进行，直到找到确切的整数解，通常迭代次数是有限的（与浮点数精度或最大可能的整数值有关）。
    
*   **空间复杂度**: `O(1)`
    我们只使用了常数个变量 `l`, `r`, `mid`, `x`, `y`, `i`, `j` 等，不需要额外的数组空间。
