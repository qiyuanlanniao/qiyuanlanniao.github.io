LeetCode 第 3613 题：[最小化连通分量的最大成本](https://leetcode.cn/problems/minimize-maximum-component-cost/description/)。
本题是一道典型的“最大值最小化”问题，其核心在于识别出答案的单调性，并将一个求解最优值的问题，巧妙地转化为一个判定性的问题，从而利用二分查找在对数时间内高效求解。
{% endcenterquote %}
<!--more-->

### 问题解读

我们得到一个包含 `n` 个节点的无向连通图，以及一个边集 `edges`，每条边都有一个权重。同时给定一个整数 `k`。

我们的任务是：通过移除图中的任意数量的边，使得最终图中最多只有 `k` 个连通分量。在所有满足这个条件的方案中，我们需要找到一个方案，使得所有连通分量中“成本”的最大值是最小的。

这里的“连通分量的成本”被定义为该分量内所有边权的最大值。如果一个分量里没有边，其成本为 0。

我们的目标就是求这个**最小的“最大成本”**。

举个例子，`n = 5`, `edges = [[0,1,4],[1,2,3],[1,3,2],[3,4,6]]`, `k = 2`。
如果我们允许的最大成本是 4，那么所有权重 > 4 的边都必须移除。在这个例子中，权重为 6 的边 `[3,4]` 被移除。剩下的边 `[0,1,4]`, `[1,2,3]`, `[1,3,2]` 形成一个连通分量 `{0, 1, 2, 3}`，其成本是 `max(4,3,2) = 4`。节点 `{4}` 自己形成一个连通分量，成本为 0。这样我们得到了 2 个连通分量，最大成本是 4，满足 `k=2` 的要求。我们可以尝试更小的最大成本，但最终会发现 4 是能达成的最小值。

### 核心思路：将“求解问题”转化为“判定问题”与二分查找

直接求解“最小的最大成本”是相当棘手的。但是，我们可以换一个角度思考：如果我们**假定**一个最大成本 `C`，我们能否判断出，通过保留所有权重 `w <= C` 的边，能否使得图中的连通分量数量不多于 `k`？

这个判定问题就清晰多了。我们可以设计一个辅助函数 `check(max_cost)` 来解决它：
1.  首先，我们假设图中所有的节点都是独立的，共有 `n` 个连通分量。
2.  我们遍历图中所有的边。
3.  只考虑那些权重 `w <= max_cost` 的边。对于每一条这样的边，我们将它连接的两个节点合并到同一个连通分量中。
4.  遍历完所有符合条件的边之后，我们统计一下图中还剩下多少个连通分量。
5.  如果连通分量的数量小于或等于 `k`，说明我们假定的这个最大成本 `max_cost` 是可行的。

为了高效地处理节点的合并与连通分量的计数，**并查集 (Disjoint Set Union, DSU)** 是最理想的数据结构。

现在，我们有了一个判定的方法。不难发现，这个判定结果具有**单调性**：
*   如果一个最大成本 `C` 是可行的（即连通分量数 `<= k`），那么任何大于 `C` 的成本 `C'` 也必然可行（因为 `C'` 会保留所有 `C` 能保留的边，以及可能更多的边，这只会让连通分量数量更少或不变）。
*   反之，如果 `C` 不可行，那么任何小于 `C` 的成本也必然不可行。

这种单调性是应用**二分查找**的完美信号。我们可以对“最大成本”这个值进行二分查找，来快速定位到那个“最小的可行成本”的临界点。

这个二分查找的搜索范围是什么呢？
*   **下界 (left)**：成本最低可以是 0，对应移除所有边的情况。
*   **上界 (right)**：成本最高不会超过所有边中的最大权重。根据题目限制，最大权重为 `10^6`。

因此，我们就在 `[0, 10^6]` 这个区间内，对最大成本进行二分查找。

### 算法详解

1.  **确定二分查找的范围**
    *   `left = 0`：可能的最低成本。
    *   `right = 10^6`：可能的最高成本。
    *   `ans` 初始化为 `right`，用来记录我们找到的可行解。

2.  **执行二分查找**
    *   当 `left <= right` 时，循环继续。
    *   计算中间值 `mid = left + (right - left) // 2` 作为当前的“猜测最大成本”。
    *   调用辅助函数 `check(mid)` 来判定这个猜测值是否可行。
        *   **如果 `check(mid)` 返回 `True`** (即 `mid` 这个成本上限是可行的)：
            *   说明 `mid` 是一个潜在的答案，我们记录下来 `ans = mid`。
            *   但是我们想找的是“最小”的最大成本，所以我们尝试一个更小的值，将搜索范围缩小到左半边：`right = mid - 1`。
        *   **如果 `check(mid)` 返回 `False`** (即 `mid` 这个成本上限太低，导致连通分量过多)：
            *   说明我们需要更大的成本上限，将搜索范围缩小到右半边：`left = mid + 1`。

3.  **返回结果**
    *   循环结束后，`ans` 中存储的就是满足条件的最小的最大成本。

### Python 代码实现

为了实现 `check` 函数，我们需要一个并查集 (DSU) 的数据结构。

```python
from typing import List

class DSU:
    """一个简单的并查集实现"""
    def __init__(self, n: int):
        # 初始化，每个节点指向自己
        self.parent = list(range(n))
        # 初始时，连通分量的数量为 n
        self.count = n

    def find(self, i: int) -> int:
        # 查找根节点，并进行路径压缩
        if self.parent[i] != i:
            self.parent[i] = self.find(self.parent[i])
        return self.parent[i]

    def union(self, i: int, j: int) -> bool:
        # 合并两个节点所在的集合
        root_i = self.find(i)
        root_j = self.find(j)
        if root_i != root_j:
            self.parent[root_i] = root_j
            # 成功合并，连通分量数量减一
            self.count -= 1
            return True
        return False

class Solution:
    def minCost(self, n: int, edges: List[List[int]], k: int) -> int:
        
        # 判定函数：在最大成本为 max_cost 的情况下，连通分量数量是否小于等于 k
        def check(max_cost: int) -> bool:
            dsu = DSU(n)
            # 遍历所有边，只使用权重 <= max_cost 的边来合并节点
            for u, v, w in edges:
                if w <= max_cost:
                    dsu.union(u, v)
            # 判断最终的连通分量数量是否满足条件
            return dsu.count <= k

        # 二分查找答案的范围
        left, right = 0, 10**6
        ans = right

        # 执行二分查找
        while left <= right:
            mid = left + (right - left) // 2
            if check(mid):
                # 如果 mid 可行，它是一个潜在答案
                # 尝试寻找更小的成本，所以向左收缩
                ans = mid
                right = mid - 1
            else:
                # 如果 mid 不可行，说明成本上限太低
                # 必须增大，所以向右收缩
                left = mid + 1
        
        return ans
```

### 复杂度分析

*   **时间复杂度**: O(E * log(W) * α(N))。
    *   `W` 是边权的最大值 (即 `10^6`)，二分查找需要 `log(W)` 次迭代。
    *   在每次迭代中，我们都需要调用 `check` 函数。该函数会遍历所有 `E` 条边。
    *   在 `check` 函数内部，每次 `union` 和 `find` 操作的平均时间复杂度接近常数，严格来说是 `α(N)`（反阿克曼函数），其中 `N` 是节点数。
    *   因此，总时间复杂度是这三者的乘积。由于 `α(N)` 增长极其缓慢，通常可视为常数，所以时间复杂度可近似为 O(E * log(W))。
*   **空间复杂度**: O(N)。
    *   主要的空间开销来自于并查集数据结构，需要一个大小为 `N` 的数组来存储父节点指针。

### 总结

本题是二分查找思想应用的又一个经典范例。它教会我们当遇到求解“最小值中的最大值”或者“最大值中的最小值”这类具有单调性的最优化问题时，可以优先考虑是否能将其转化为一个更简单的“判定性”问题。如果可以，那么二分查找就是解决这类问题的绝佳工具，它能将搜索范围不断折半，从而在对数级别的时间复杂度内找到最优解。
