LeetCode 第 778 题：[水位上升的泳池中游泳](https://leetcode.cn/problems/swim-in-water/description/)。
当直接求解“最小的等待时间”比较困难时，可以通过二分查找猜测一个时间，然后验证这个时间是否可行，从而将一个求解最值的问题转化为一个判定性的问题。
{% endcenterquote %}
<!--more-->

### 问题解读

在一个 `n x n` 的网格中，每个格子 `grid[i][j]` 代表了该位置平台的高度。随着时间 `t` 的流逝，水位也会上升到 `t`。我们可以从一个平台游到相邻的另一个平台，前提是这两个平台的高度都必须小于或等于当前的水位 `t`。

我们的目标是从左上角 `(0, 0)` 出发，到达右下角 `(n-1, n-1)`。我们需要计算出，为了能够成功到达终点，所需要等待的**最少时间**是多少。

举个例子，`grid = [[0,2],[1,3]]`。
*   在时间 `t=0`、`t=1` 或 `t=2` 时，我们都无法从 `(0,0)`（高度0）到达任何一个相邻平台，因为 `(0,1)` 的高度是2，`(1,0)` 的高度是1。例如，在 `t=1` 时，虽然可以淹没 `(0,0)` 和 `(1,0)`，但无法淹没 `(0,1)`，因此没有通路。
*   直到时间 `t=3` 时，水位为3，淹没了所有的平台。此时，从 `(0,0)` 到 `(n-1,n-1)` 之间必然存在一条通路。

所以，我们需要找到那个**恰好**能让起点和终点连通的最小时间点。

### 核心思路：将“求解问题”转化为“判定问题”与二分查找

直接求解“最少时间”似乎没有清晰的思路。但是，我们可以换一个角度来思考这个问题：如果我们**假定**一个时间 `t`，我们能否判断出，在水位为 `t` 的情况下，是否存在一条从 `(0, 0)` 到 `(n-1, n-1)` 的路径？

这个“判定问题”就相对简单了。我们可以构建一个辅助函数 `check(t)`：
*   该函数接收一个时间 `t` 作为参数。
*   在网格中，所有高度 `grid[i][j] <= t` 的平台都可以被视为通路，而高度大于 `t` 的平台则是障碍。
*   问题就变成了：在一个图中，判断从起点 `(0, 0)` 到终点 `(n-1, n-1)` 是否存在一条路径。
*   我们可以使用深度优先搜索（DFS）或广度优先搜索（BFS）来轻松解决这个连通性问题。

有了这个判定函数后，我们观察到它具有非常明显的**单调性**：
*   如果时间 `t` **可以**让起点和终点连通，那么任何大于 `t` 的时间 `t'`（水位更高）也**一定**可以连通。
*   反之，如果时间 `t` **不可以**让起点和终点连通，那么任何小于 `t` 的时间 `t''`（水位更低）也**必然**无法连通。

这种单调性是应用**二分查找**的完美信号。我们可以对“时间 `t`”这个值进行二分查找，来快速定位到那个“最小可行”的临界点。

这个二分查找的搜索范围是什么呢？
*   **下界 (left)**：我们至少要等到起点 `(0,0)` 和终点 `(n-1, n-1)` 本身被水淹没，所以时间 `t` 的最小值至少是 `max(grid[0][0], grid[n-1][n-1])`。
*   **上界 (right)**：当时间 `t` 等于网格中平台的最大可能高度时，所有的平台都被淹没，此时必然是连通的。因为 `0 <= grid[i][j] < n*n`，所以上界可以取 `n*n - 1`。

因此，我们就在 `[max(grid[0][0], grid[n-1][n-1]), n*n - 1]` 这个区间内，对时间 `t` 进行二分查找。

### 算法详解

1.  **确定二分查找的范围**
    *   `left = max(grid[0][0], grid[n-1][n-1])`：这是可能的最低可行时间。
    *   `right = n*n - 1`：这是必然可行的最高时间。

2.  **执行二分查找**
    *   当 `left < right` 时，循环继续。
    *   计算中间值 `mid = left + (right - left) // 2` 作为当前的“猜测时间”。
    *   调用辅助函数 `check(mid)` 来判定这个猜测值是否可行。
        *   `check(mid)` 函数内部，使用 DFS 进行路径搜索：
            *   创建一个 `visited` 集合，防止重复访问。
            *   从 `(0, 0)` 开始递归搜索。
            *   如果当前平台 `(i, j)` 的高度 `grid[i][j] > mid`，则此路不通。
            *   否则，标记 `(i, j)` 已访问，并向其四周（上、下、左、右）未访问过的邻居递归。
            *   如果 DFS 过程中能成功到达 `(n-1, n-1)`，则 `check(mid)` 返回 `True`。
        *   **如果 `check(mid)` 返回 `True`** (即时间 `mid` 足够到达终点)：
            *   说明 `mid` 是一个潜在的答案，但我们想找的是“最少”时间，所以我们尝试一个更小的值，将搜索范围缩小到左半边（包含 mid）：`right = mid`。
        *   **如果 `check(mid)` 返回 `False`** (即时间 `mid` 还不够)：
            *   说明我们需要更多的时间，将搜索范围缩小到右半边：`left = mid + 1`。

3.  **返回结果**
    *   循环结束后，`left` 和 `right` 相遇，此时的值就是满足条件的最低等待时间。

### Python 代码实现

```python
from typing import List

class Solution:
    def swimInWater(self, grid: List[List[int]]) -> int:
        n = len(grid)

        # 判定函数：在时间 t (水位为 t) 时，是否可以从 (0,0) 到达 (n-1, n-1)
        def check(t: int) -> bool:
            # 起点的高度如果都大于 t，直接判定为不可达
            if grid[0][0] > t:
                return False
            
            visited = set()
            
            # 使用深度优先搜索 (DFS) 检查连通性
            def dfs(i: int, j: int) -> bool:
                # 到达终点
                if i == n - 1 and j == n - 1:
                    return True
                
                visited.add((i, j))
                
                # 探索四个方向
                for x, y in [(i - 1, j), (i, j + 1), (i + 1, j), (i, j - 1)]:
                    # 检查新坐标是否在网格内、未被访问过、且高度不超过当前水位 t
                    if 0 <= x < n and 0 <= y < n and (x, y) not in visited and grid[x][y] <= t:
                        if dfs(x, y):
                            return True
                            
                return False

            return dfs(0, 0)

        # 二分查找的左右边界
        # 左边界：至少要能淹没起点和终点
        # 右边界：最大可能的高度，此时一定可达
        left, right = max(grid[0][0], grid[n-1][n-1]), n * n - 1
        
        # 二分查找寻找最小的可行时间 t
        while left < right:
            mid = left + (right - left) // 2
            if check(mid):
                # 如果 mid 时间可行，说明它可能是答案，
                # 或者还有更小的可行时间，所以我们尝试在左半部分 [left, mid] 继续寻找
                right = mid
            else:
                # 如果 mid 时间不可行，说明时间太少，
                # 必须增加，所以去右半部分 [mid + 1, right] 寻找
                left = mid + 1
        
        # 循环结束时，left 就是满足条件的最低时间
        return left
```

### 复杂度分析

*   **时间复杂度**: O(N² * log(N²))。
    *   二分查找的搜索空间大小为 `n*n`，因此迭代次数为 O(log(N²))，即 O(log(N))。
    *   在每次迭代中，我们都需要调用 `check` 函数。`check` 函数内部的 DFS/BFS 最多会访问网格中的 `N*N` 个单元格一次。
    *   因此，总时间复杂度是这两者的乘积，即 O(N² * log(N²))。
*   **空间复杂度**: O(N²)。
    *   空间开销主要来自于 `check` 函数中的 `visited` 集合和 DFS 的递归调用栈。在最坏的情况下，两者都需要存储 O(N²) 个元素。

### 总结

本题是二分查找思想应用的又一经典范例。它展示了当直接求解最优值（如“最小值”、“最大值”）困难时，如何通过转换思维，将其变为一个更简单的“判定性”问题。如果这个判定问题随输入参数的变化呈现出单调性，那么二分查找就是解决这类问题的强大武器。它能将搜索范围不断折半，从而在对数级别的时间复杂度内高效地逼近最优解。
