LeetCode 第 1642 题：[可以到达的最远建筑](https://leetcode.cn/problems/furthest-building-you-can-reach/description/)。
本题的核心在于识别出“能否到达某个建筑”这一判定的单调性，从而将一个求解最大值的问题，巧妙地转化为一个判定性的问题，并利用二分查找高效求解。
{% endcenterquote %}
<!--more-->

### 问题解读

我们需要在一系列高度不同的建筑物之间移动，从第 0 个建筑出发，尽可能到达最远的建筑。我们拥有一定数量的砖块 `bricks` 和梯子 `ladders` 作为辅助工具。

核心的移动规则是：
1.  从建筑 `i` 到 `i+1`，如果 `heights[i] >= heights[i+1]`（即向下或平着走），不需要任何工具。
2.  如果 `heights[i] < heights[i+1]`（即向上爬），则需要克服 `h[i+1] - h[i]` 的高度差。此时，我们可以选择使用相应数量的砖块，或者使用一架梯子。梯子可以克服任意高度差。

我们的目标是在砖块和梯子的约束下，找到能到达的**最远**的建筑下标。

举个例子，`heights = [4,2,7,6,9,14,12]`, `bricks = 5`, `ladders = 1`。
- 到达建筑 1 (高度2): 向下走，无消耗。
- 到达建筑 2 (高度7): 向上爬，高度差为 5。可以使用 5 个砖块。剩余 `bricks=0`, `ladders=1`。
- 到达建筑 3 (高度6): 向下走，无消耗。
- 到达建筑 4 (高度9): 向上爬，高度差为 3。此时砖块用完，只能使用梯子。剩余 `bricks=0`, `ladders=0`。
- 尝试到达建筑 5 (高度14): 向上爬，高度差为 5。此时砖块和梯子都已耗尽，无法继续前进。
因此，能到达的最远建筑是下标 4。

### 核心思路：将“求解问题”转化为“判定问题”与二分查找

直接计算能到达的最远距离似乎很复杂，因为在每个需要向上爬的地方，我们都要做出“用砖块还是用梯子”的决策。

一个更巧妙的思路是转变问题：与其直接问“最远能到哪”，不如问一个更容易判断的问题：“我们**能否到达**第 `k` 个建筑？”

这个问题相对简单。我们可以写一个辅助函数 `can_reach(k)` 来进行判断。为了判断能否到达建筑 `k`，我们需要考虑从建筑 0 到 `k` 的所有需要向上爬的路径，并以最节省的方式使用资源。显然，梯子比砖块更宝贵，因为它能应对无限高的高度差。因此，最优策略是**将有限的梯子用在最高的几次攀爬上**，而用砖块来应对相对较小的高度差。

这个判定策略具有非常重要的**单调性**：
*   如果我们可以到达建筑 `k`，那么对于任何小于 `k` 的建筑 `j`，我们必然也能到达。
*   反之，如果我们无法到达建筑 `k`，那么我们也绝不可能到达任何比 `k` 更远的建筑。

这种单调性是应用二分查找的完美信号。 我们可以对“建筑的下标”进行二分查找，来快速定位到那个“最远可到达”的临界点。

这个二分查找的搜索范围是什么呢？
*   **下界 (left)**：`0`，即起点。
*   **上界 (right)**：`len(heights) - 1`，即最后一个建筑。

因此，我们就在 `[0, len(heights) - 1]` 这个区间内，对建筑下标进行二分查找。

### 算法详解

1.  **确定二分查找的范围**
    *   `left = 0`：可能的最小答案。
    *   `right = len(heights) - 1`：可能的最大答案。
    *   `ans` 初始化为 `0`，用来记录我们找到的可行解。

2.  **执行二分查找**
    *   当 `left <= right` 时，循环继续。
    *   计算中间值 `mid = left + (right - left) // 2` 作为当前的“猜测能到达的建筑下标”。
    *   调用辅助函数 `check(mid)` 来判定这个猜测值是否可行。
        *   **`check(mid)` 的内部逻辑**：
            1. 收集从建筑 0 到 `mid` 之间所有需要向上爬的高度差，存入列表 `climbs`。
            2. 如果 `climbs` 的数量小于等于 `ladders`，说明所有攀爬都可以用梯子解决，返回 `True`。
            3. 否则，对 `climbs` 进行排序。我们将梯子用于 `ladders` 个最大的高度差。
            4. 计算剩下较小的高度差之和，即为需要的砖块数。
            5. 如果所需砖块数不大于我们拥有的 `bricks`，返回 `True`；否则返回 `False`。
        *   **如果 `check(mid)` 返回 `True`**：
            *   说明 `mid` 是一个可以到达的位置，我们记录下来 `ans = mid`。
            *   既然 `mid` 可以到达，我们应该尝试走得更远，将搜索范围缩小到右半边：`left = mid + 1`。
        *   **如果 `check(mid)` 返回 `False`**：
            *   说明 `mid` 到不了，那更远的地方也肯定到不了。我们将搜索范围缩小到左半边：`right = mid - 1`。

3.  **返回结果**
    *   循环结束后，`ans` 中存储的就是能到达的最远建筑物的下标。

### Python 代码实现

```python
from typing import List

class Solution:
    def furthestBuilding(self, heights: List[int], bricks: int, ladders: int) -> int:
        n = len(heights)

        # 辅助函数：检查是否能到达第 k 个建筑
        def check(k: int) -> bool:
            if k == 0:
                return True
            
            # 收集从 0 到 k 需要攀爬的高度差
            climbs = []
            for i in range(k):
                diff = heights[i+1] - heights[i]
                if diff > 0:
                    climbs.append(diff)
            
            # 如果攀爬次数不多于梯子数，肯定可以到达
            if len(climbs) <= ladders:
                return True
            
            # 对高度差进行排序，优先用梯子处理最大的几次攀爬
            climbs.sort()
            
            # 剩下的攀爬需要用砖块
            # 我们需要为最小的 len(climbs) - ladders 次攀爬提供砖块
            bricks_needed = sum(climbs[:len(climbs) - ladders])
            
            return bricks_needed <= bricks

        # 二分查找最远的 k
        left, right = 0, n - 1
        ans = 0
        while left <= right:
            mid = left + (right - left) // 2
            if check(mid):
                # 如果 mid 可以到达，它就是一个潜在的答案
                # 我们继续向右搜索，看能否走得更远
                ans = mid
                left = mid + 1
            else:
                # 如果 mid 无法到达，说明目标太远了，需要向左搜索
                right = mid - 1
        
        return ans
```

### 复杂度分析

*   **时间复杂度**: O(N * log(N) * log(N))。
    *   `N` 是建筑物数组 `heights` 的长度。
    *   二分查找本身需要 `log(N)` 次迭代。
    *   在每次迭代中，我们调用 `check(mid)` 函数。该函数内部：
        *   遍历 `mid` 次来收集高度差，时间为 O(mid)。
        *   对最多 `mid` 个高度差进行排序，时间为 O(mid * log(mid))。
    *   因此，`check(mid)` 的时间复杂度由排序主导，为 O(mid * log(mid))。
    *   总时间复杂度近似为 `log(N)` 乘以 `check` 函数在 `mid` 接近 `N` 时的复杂度，即 O(log(N) * N * log(N))。
*   **空间复杂度**: O(N)。
    *   在 `check` 函数中，我们需要一个列表来存储所有的高度差，其大小最多为 `N-1`。

### 总结

本题是“二分答案”这一经典算法模式的绝佳体现。 当遇到求解“最大值”或“最小值”这类优化问题时，如果可以直接判断某个潜在答案是否可行，并且这个可行性判断具有单调性，那么就可以考虑使用二分查找。 这种方法将复杂的求解问题转化为一系列简单的判定问题，从而大幅优化算法效率，是一种非常强大和常用的解题技巧。
