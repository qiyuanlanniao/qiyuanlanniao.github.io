LeetCode 第 3143 题：[正方形中的最多点数](https://leetcode.cn/problems/maximum-points-inside-the-square/description/)。
本题的核心在于识别出答案关于正方形大小的单调性，并将一个求解最大值的问题，巧妙地转化为一个判定性的问题，从而利用二分查找在对数时间内高效求解。
{% endcenterquote %}
<!--more-->

### 问题解读

我们需要在一个二维平面上找到一个“合法”的正方形，使其能包含尽可能多的点。

这里的核心约束条件是：
1.  正方形必须以原点 `(0, 0)` 为中心，且所有边都平行于坐标轴。
2.  一个正方形是“合法的”，当且仅当其内部（包括边界）不包含任何具有相同标签的两个点。
3.  一个点 `(x, y)` 如果在边长为 `2 * size` 的正方形内，则必须满足 `abs(x) <= size` 且 `abs(y) <= size`。

我们的目标是找出所有合法正方形中，能够包含的点的数量的最大值。

举个例子，`points = [[2,2],[-1,-2],[-4,4],[-3,1],[3,-3]]`, `s = "abdca"`。
如果一个正方形的半边长 `size` 为 3，它会包含 `[2,2]` (a), `[-1,-2]` (b), `[-3,1]` (d), `[3,-3]` (c)。这些点的标签 `a, b, d, c` 互不相同，所以这是一个合法的正方形，包含 4 个点。
如果 `size` 增大到 4，`[-4,4]` (a) 点也被包含进来。现在正方形内有了 `[2,2]` (a) 和 `[-4,4]` (a)，出现了重复标签 `a`，因此这个正方形是“不合法”的。

这暗示了正方形的合法性与其大小之间存在一种单调关系：如果一个大小为 `size` 的正方形不合法，那么任何比它更大的正方形也必然不合法。

我们的任务就是在所有合法的正方形尺寸中，找到那个能容纳最多点数的尺寸，并返回其容纳的点数。

### 核心思路：将“求解问题”转化为“判定问题”与二分查找

直接求解“最多点数”似乎无从下手，因为点数并不直接与正方形大小线性相关。但是，我们可以换一个角度：对正方形的半边长 `size` 进行分析。

我们可以将问题转化为一个更容易判断的“判定问题”：**给定一个半边长 `size`，判断由它构成的正方形是否合法？**

这个判定问题很容易解决。我们可以写一个辅助函数 `check(size)`，来模拟判断过程：
*   初始化一个空的哈希集合 `visited_tags`，用来存放当前正方形内所有点的标签。
*   遍历所有点 `(x, y)` 和其对应的标签 `c`：
    *   如果这个点在正方形内部，即 `abs(x) <= size` 且 `abs(y) <= size`：
        *   检查它的标签 `c` 是否已经在 `visited_tags` 中。
        *   如果已存在，说明出现了重复标签，该 `size` 构成的正方形**不合法**，函数返回 `True`（表示“有问题”）。
        *   如果不存在，将标签 `c` 加入 `visited_tags`。
*   如果遍历完所有点都没有发现重复标签，说明该 `size` 构成的正方形是**合法的**，函数返回 `False`（表示“没问题”）。

现在，我们有了一个判定方法。并且可以清晰地看到，这个判定结果具有**单调性**：
*   如果一个大小为 `size` 的正方形不合法（`check(size)` 返回 `True`），那么任何更大的 `size' > size` 也必然不合法，因为它会包含所有导致 `size` 不合法的点。
*   反之，如果 `size` 合法（`check(size)` 返回 `False`），那么比它更小的尺寸也可能合法。

这种单调性是应用二分查找的完美信号。我们可以对“正方形的半边长 `size`”进行二分查找，来快速定位到那个“从合法变为不合法”的临界点。

这个二分查找的搜索范围是什么呢？
*   **下界 (left)**：`0`，一个大小为 0 的正方形。
*   **上界 (right)**：`10^9 + 1`，比题目中坐标的最大绝对值还要大，确保能包含所有点。

我们二分的目标是找到**第一个不合法的 `size`**。那么，比这个临界值小 1 的 `size - 1`，就是最大的那个合法尺寸。

### 算法详解

1.  **确定二分查找的范围**
    *   `left = 0`
    *   `right = 10^9 + 1`
    *   我们的目标是找到满足 `check(size)` 为 `True` 的最小值。

2.  **执行二分查找**
    *   当 `left < right` 时，循环继续。
    *   计算中间值 `mid = left + (right - left) // 2` 作为当前的“猜测半边长”。
    *   调用辅助函数 `check(mid)` 来判定这个猜测值是否可行。
        *   **如果 `check(mid)` 返回 `True`** (即 `mid` 尺寸的正方形**不合法**)：
            *   说明 `mid` 本身或更小的值才是我们寻找的临界点。我们将搜索范围缩小到左半边（包含 `mid`）：`right = mid`。
        *   **如果 `check(mid)` 返回 `False`** (即 `mid` 尺寸的正方形**合法**)：
            *   说明这个尺寸是安全的，临界点一定在它右边。我们将搜索范围缩小到右半边：`left = mid + 1`。

3.  **返回结果**
    *   循环结束后，`left` (或 `right`) 指向的就是第一个不合法的尺寸。
    *   因此，最大的合法尺寸就是 `left - 1`。
    *   最后，我们只需计算在 `size = left - 1` 的正方形内有多少个点即可。
    *   （在题目给出的精妙实现中，点数的计算被巧妙地整合进了 `check` 函数，避免了最后一次的重复计数）。

### Python 代码实现

下面是题解中给出的代码，它利用了二分查找，并将最终结果的计算嵌入到了检查函数中，非常高效。

```python
from bisect import bisect_left
from typing import List

class Solution:
    def maxPointsInsideSquare(self, points: List[List[int]], s: str) -> int:
        # ans 用于存储在最后一次“合法”检查中，正方形内的点数
        ans = 0

        # 判定函数：检查半边长为 size 的正方形是否“不合法”（即有重复标签）
        # 返回 True 表示不合法，False 表示合法
        def check(size: int) -> bool:
            vis = set()
            count = 0
            for (x, y), c in zip(points, s):
                # 检查点是否在正方形内
                if abs(x) <= size and abs(y) <= size:
                    if c in vis:
                        # 发现重复标签，此 size 不合法
                        return True
                    vis.add(c)
                    count += 1
            
            # 如果循环正常结束，说明此 size 合法
            # 用 nonlocal/self.ans 更新当前合法尺寸下的点数
            nonlocal ans
            ans = count
            return False

        # 在 [0, 10^9+1] 的范围内二分查找第一个使 check 返回 True (不合法) 的 size
        # bisect_left 的 key 函数返回一个布尔值序列，如 [F, F, F, T, T, T]
        # 它会找到第一个 T 的索引
        invalid_size = bisect_left(range(10**9 + 1), True, key=check)
        
        # 当 check(invalid_size - 1) 被调用时，它是最后一次返回 False 的调用，
        # 那时 ans 会被正确更新为最大合法正方形内的点数。
        # 如果最小的 size 都不合法（例如第一个点就有重复标签），
        # 那么 check(0) 就会返回 True，bisect_left 结果是 0，ans 初始值 0 也是正确的。
        return ans

```

### 复杂度分析

*   **时间复杂度**: O(N * log(C))。
    *   `N` 是点的数量。
    *   `C` 是坐标的取值范围，即 `10^9`。
    *   二分查找本身需要 `log(C)` 次迭代。
    *   在每次迭代中，我们都需要调用 `check` 函数，该函数需要遍历一次 `points` 数组，时间复杂度为 O(N)。
    *   因此，总时间复杂度是这两者的乘积。
*   **空间复杂度**: O(A)。
    *   `A` 是字符集的大小，这里是 26。
    *   `check` 函数中 `vis` 集合最多存储 26 个小写字母，因此空间复杂度是常数级的。

### 总结

本题是二分查找思想应用的经典范例。它教会我们当遇到求解最优化问题（最大/最小值）时，可以检查答案是否具有单调性。如果满足，就可以尝试将其转化为一个更简单的“判定性”问题。通过对答案的可能范围进行二分，我们可以将搜索空间不断折半，从而在对数级别的时间复杂度内找到最优解，这比线性扫描所有可能的答案要高效得多。
