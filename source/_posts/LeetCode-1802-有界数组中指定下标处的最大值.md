LeetCode 第 1802 题：[有界数组中指定下标处的最大值](https://leetcode.cn/problems/maximum-value-at-a-given-index-in-a-bounded-array/description/)。
本题的核心在于识别出答案的单调性，将一个求解“最大值”的问题，巧妙地转化为一个“可行性判定”的问题，从而利用二分查找在对数时间内高效求解。
{% endcenterquote %}
<!--more-->

### 问题解读

我们需要根据给定的三个正整数 `n`、`index` 和 `maxSum`，构造一个数组 `nums`，并找到 `nums[index]` 可能的最大值。

这个构造过程需要满足以下几个核心约束条件：
1.  数组长度为 `n`。
2.  数组中的每个元素 `nums[i]` 都必须是正整数（即 `nums[i] >= 1`）。
3.  相邻元素之差的绝对值不能超过 1，即 `abs(nums[i] - nums[i+1]) <= 1`。
4.  数组所有元素的总和不能超过 `maxSum`。
5.  在满足以上所有条件的前提下，让 `nums[index]` 的值尽可能大。

我们的目标就是找出这个被最大化的 `nums[index]`。

这个问题的有趣之处在于第三个条件 `abs(nums[i] - nums[i+1]) <= 1`。为了在总和 `maxSum` 的限制下尽可能地增大 `nums[index]` 的值，我们直觉上应该让其他位置的元素尽可能小。这个约束意味着，数组的值会以 `index` 为中心，向两侧逐渐递减，每次最多减 1，直到减至 1 为止，形成一个“山峰”形状。例如，如果 `nums[index] = 3`，它旁边的元素最大只能是 2 或 3，为了总和最小，我们选择 2，再旁边就是 1，之后的元素都会保持为 1。

### 核心思路：将“求解问题”转化为“判定问题”与二分查找

直接计算出 `nums[index]` 的最大值非常困难。但是，我们可以换一个角度思考：如果我们**假设** `nums[index]` 的值是一个具体的数 `v`，我们能否判断出是否存在一个满足所有条件的数组？

这个问题就相对简单了。我们可以编写一个辅助函数 `check(v)`，来判断当 `nums[index] = v` 时，构造出的数组其*最小可能总和*是否会超过 `maxSum`。
*   为了使总和最小，以 `nums[index] = v` 为峰值，数组其他元素向两侧依次递减 1，直到达到 1 为止。
*   剩下的所有位置都用 1 填充。
*   我们计算出这个“山峰”数组的总和。如果这个最小总和不大于 `maxSum`，那就说明 `v` 是一个**可行**的值。

现在，我们有了一个判定的方法。我们可以观察到这个判定结果具有**单调性**：
*   如果 `nums[index]` 取值为 `v` 时是可行的（即最小总和 `≤ maxSum`），那么任何小于 `v` 的值 `v'` 也一定是可行的，因为 `v'` 对应的最小总和只会更小。
*   反之，如果 `v` 是不可行的（最小总和 `> maxSum`），那么任何大于 `v` 的值也必然不可行。

这种单调性是应用二分查找的绝佳信号。我们可以对 `nums[index]` 的可能取值进行二分查找，来快速定位到那个“最大可行”的临界点。

这个二分查找的搜索范围是什么呢？
*   **下界 (left)**：数组中每个元素至少为 1，所以 `nums[index]` 的最小值是 `1`。
*   **上界 (right)**：在最极端的情况下，`nums[index]` 的值也不会超过 `maxSum`，所以上界可以设为 `maxSum`。

因此，我们就在 `[1, maxSum]` 这个区间内，对 `nums[index]` 的值进行二分查找。

### 算法详解

1.  **确定二分查找的范围**
    *   `left = 1`：`nums[index]` 可能的最小值。
    *   `right = maxSum`：`nums[index]` 可能的最大值。
    *   `ans` 用来记录我们找到的可行解。

2.  **执行二分查找**
    *   当 `left <= right` 时，循环继续。
    *   计算中间值 `mid = left + (right - left) // 2` 作为当前的“猜测值”。
    *   调用辅助函数 `check(mid)` 来判定这个猜测值是否可行。该函数计算当 `nums[index] = mid` 时，数组的最小总和。
        *   **如果 `check(mid)` 返回 `True`** (即最小总和 `≤ maxSum`)：
            *   说明 `mid` 是一个潜在的答案，我们记录下来 `ans = mid`。
            *   但是我们想找的是“最大”的值，所以我们尝试一个更大的值，将搜索范围缩小到右半边：`left = mid + 1`。
        *   **如果 `check(mid)` 返回 `False`** (即最小总和 `> maxSum`)：
            *   说明 `mid` 这个值太大了，我们需要更小的值，将搜索范围缩小到左半边：`right = mid - 1`。

3.  **返回结果**
    *   循环结束后，`ans` 中存储的就是满足条件的最大值。

#### `check(v)` 函数的实现细节
为了计算当 `nums[index] = v` 时的最小和，我们可以分别计算 `index` 左侧和右侧的和，最后再加上 `v` 本身（注意避免重复计算）。

*   **左侧部分**（从 `0` 到 `index`，共 `index + 1` 个元素）：
    *   如果峰值 `v` 大于左侧长度 `index + 1`，则数组形态为 `[v-index, ..., v-1, v]`，这是一个等差数列。
    *   如果峰值 `v` 小于等于左侧长度，则数组形态为 `[1, ..., 1, 2, ..., v]`，它由 `(index + 1) - v` 个 1 和一个从 1 到 `v` 的等差数列组成。
*   **右侧部分**（从 `index` 到 `n-1`，共 `n - index` 个元素）：
    *   逻辑与左侧类似，根据 `v` 和右侧长度 `n - index` 的大小关系来计算。

最后将左侧和与右侧和相加，并减去被重复计算的峰值 `v`，即可得到整个数组的最小总和。

### Python 代码实现

```python
class Solution:
    def maxValue(self, n: int, index: int, maxSum: int) -> int:
        """
        使用二分查找来寻找 nums[index] 的最大可能值。
        """
        
        def check(v: int) -> bool:
            """
            辅助函数，用于检查当 nums[index] = v 时，
            构造出的数组其最小和是否小于等于 maxSum。
            """
            # 左侧部分（从 0 到 index）的长度和最小和
            left_len = index + 1
            left_sum = 0
            if v > left_len:
                # 数组形态为 [v-index, ..., v-1, v]
                # 等差数列求和：(首项 + 末项) * 项数 / 2
                left_sum = (v + v - index) * left_len // 2
            else:
                # 数组形态为 [1, ..., 1, 2, ..., v]
                # 1到v的和 + 剩余的1
                left_sum = v * (v + 1) // 2 + (left_len - v)
            
            # 右侧部分（从 index 到 n-1）的长度和最小和
            right_len = n - index
            right_sum = 0
            if v > right_len:
                # 数组形态为 [v, v-1, ..., v-(right_len-1)]
                right_sum = (v + v - (right_len - 1)) * right_len // 2
            else:
                # 数组形态为 [v, v-1, ..., 1, ..., 1]
                right_sum = v * (v + 1) // 2 + (right_len - v)

            # 总和为左侧和 + 右侧和 - 重复计算的峰值 v
            total_sum = left_sum + right_sum - v
            
            return total_sum <= maxSum

        # 对 nums[index] 的可能值进行二分查找
        left, right = 1, maxSum
        ans = 0
        while left <= right:
            mid = (left + right) // 2
            if check(mid):
                # mid 是一个可行解，尝试寻找更大的解
                ans = mid
                left = mid + 1
            else:
                # mid 太大，需要缩小范围
                right = mid - 1
        
        return ans

```

### 复杂度分析

*   **时间复杂度**: O(log(maxSum))。
    *   二分查找的搜索空间大小为 `maxSum`。
    *   在每次二分迭代中，我们调用的 `check` 函数内部只进行常数次算术运算，其时间复杂度为 O(1)。
    *   因此，总时间复杂度由二分查找的次数决定。
*   **空间复杂度**: O(1)。
    *   除了输入参数外，我们只使用了几个变量来存储左右边界和中间值，所以空间复杂度是常数级的。

### 总结

本题是“二分答案”思想的又一个经典应用。当直接求解一个“最大值”或“最小值”问题比较困难时，可以思考能否将其转化为一个更简单的“判定性”问题。如果这个判定问题对于答案具有单调性，那么就可以利用二分查找，将搜索范围不断折半，从而在对数级别的时间复杂度内找到最优解，这比线性扫描所有可能的答案要高效得多。
