LeetCode 第 668 题：[乘法表中第k小的数](https://leetcode.cn/problems/kth-smallest-number-in-multiplication-table/description/)。
本题可以转化为一个判定性问题：给定整数 x，统计乘法表中 ≤x 的元素个数 cnt，判断是否满足 cnt≥k。
{% endcenterquote %}
<!--more-->

### 问题描述

给定一个 `m x n` 的乘法表，以及一个正整数 `k`，你需要返回表中第 `k` 小的数字。

乘法表是一个整数矩阵 `mat`，其中 `mat[i][j] == i * j`（下标从 1 开始）。

**示例 1：**

```
输入：m = 3, n = 3, k = 5
输出：3
解释：
乘法表:
1	2	3
2	4	6
3	6	9

排序后的序列为：1, 2, 2, 3, 3, 4, 6, 6, 9。第 5 小的数字是 3。
```

**示例 2：**
```
输入：m = 2, n = 3, k = 6
输出：6
解释：
乘法表:
1	2	3
2	4	6

排序后的序列为：1, 2, 2, 3, 4, 6。第 6 小的数字是 6。
```

**提示：**
*   `1 <= m, n <= 3 * 10^4`
*   `1 <= k <= m * n`

### 解题思路

#### 暴力解法为何不可行？

一个直观的想法是生成整个 `m x n` 的乘法表，将其展平成一个一维数组，然后排序，最后返回第 `k` 个元素。

然而，根据题目的约束条件，`m` 和 `n` 的最大值都可以达到 30000。这意味着乘法表最多可能有 `30000 * 30000 = 9 * 10^8` 个元素。 创建和存储如此巨大的数组会导致内存溢出，因此暴力解法是不可行的。

#### 二分答案

这道题是“寻找第 k 小元素”的经典变体。对于这类问题，一个强大的思想是**二分答案**。

我们要求的解（第 k 小的数）一定在 `[1, m*n]` 这个区间内。我们可以对这个数值区间进行二分查找。假设我们猜测第 `k` 小的数是 `mid`。为了验证这个猜测，我们需要快速计算出乘法表中有多少个数小于等于 `mid`。我们把这个计数函数记为 `count(mid)`。

*   如果 `count(mid) >= k`，这说明 `mid` 太大了，或者 `mid` 正好是我们要找的答案。真正的第 `k` 小的数可能更小，或者就是 `mid`。因此，我们应该在 `[1, mid]` 这个区间继续查找。
*   如果 `count(mid) < k`，这说明 `mid` 太小了，乘法表中比它小的数不足 `k` 个。因此，第 `k` 小的数一定比 `mid` 大。我们应该在 `[mid + 1, m*n]` 这个区间继续查找。

通过不断缩小范围，我们最终可以找到满足条件的最小的那个数，即为第 `k` 小的数。

#### 如何实现 `count(x)` 函数？

现在问题的核心变成了如何高效地计算乘法表里小于等于 `x` 的数的个数。

乘法表的每一行 `i` （`1 <= i <= m`）都是一个等差数列：`1*i, 2*i, 3*i, ..., n*i`。

对于第 `i` 行，我们要计算有多少个 `j*i <= x`，其中 `1 <= j <= n`。
这等价于 `j <= x / i`。
所以，在第 `i` 行，小于等于 `x` 的数的个数就是 `floor(x / i)`。但是，这个个数不能超过该行的总列数 `n`。

因此，第 `i` 行中小于等于 `x` 的数字个数为 `min(floor(x / i), n)`。

我们将每一行的计数加起来，就得到了整个乘法表中不大于 `x` 的数字总数：
`count(x) = sum(min(floor(x / i), n) for i in 1..m)`

### 代码实现

```python
class Solution:
    def findKthNumber(self, m: int, n: int, k: int) -> int:
        """
        使用二分查找在答案空间 [1, m*n] 中寻找第 k 小的数。
        """
        
        def check(x: int) -> bool:
            """
            检查乘法表中小于等于 x 的数的个数是否大于等于 k。
            """
            count = 0
            # 遍历每一行
            for i in range(1, m + 1):
                # 对于第 i 行，其元素为 i, 2*i, 3*i, ...
                # 小于等于 x 的元素个数为 x // i，但不能超过列数 n
                count += min(x // i, n)
            return count >= k

        # 答案的范围是 [1, m*n]
        left, right = 1, m * n
        ans = 0

        while left <= right:
            mid = (left + right) // 2
            if check(mid):
                # mid 可能是答案，或者答案在 mid 的左侧
                # 我们需要找到满足条件的最小的 mid
                ans = mid
                right = mid - 1
            else:
                # mid 太小了，答案在 mid 的右侧
                left = mid + 1
        
        return ans
```

### 复杂度分析

*   **时间复杂度**: `O(m * log(m*n))`。二分查找的范围是 `m*n`，所以对数项是 `log(m*n)`。在每次二分检查中，`check` 函数需要 `O(m)` 的时间来遍历 `m` 行。
*   **空间复杂度**: `O(1)`。我们只使用了常数个变量。
