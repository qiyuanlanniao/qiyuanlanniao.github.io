LeetCode 第 395 题：[至少有 K 个重复字符的最长子串](https://leetcode.cn/problems/longest-substring-with-at-least-k-repeating-characters/)。
此题要求在一个字符串中找到一个最长的子串，该子串中每个字符的出现次数都不少于 K。条件的“非单调性”让常规SW思路失效。我们需要换个角度，通过枚举一个额外约束来让滑动窗口重新焕发生机。
{% endcenterquote %}
<!--more-->
### 问题解读

题目给定一个字符串 `s` 和一个整数 `k`，要求我们找到 `s` 中最长的一个子串，这个子串有一个特殊的性质：子串中出现的**每一个**字符，其出现次数都必须**不少于** `k`。最后返回这个最长子串的长度。

举个例子，对于 `s = "ababbc"`, `k = 2`：
*   子串 "ababb" 是一个满足条件的子串。其中 'a' 出现了 2 次，'b' 出现了 3 次，都不少于 `k=2`。其长度为 5。
*   子串 "ababbc" 则不满足条件，因为 'c' 只出现了 1 次，小于 `k=2`。
*   "ababb" 是能找到的最长的满足条件的子串，因此答案是 5。

如果找不到这样的子串，则返回 0。

### 核心思路：枚举+滑动窗口

直接解决这个问题的一个难点在于，一个“不满足条件”的窗口，无论向右扩大还是向左收缩，都无法保证其会趋向于“满足条件”。例如，一个窗口因为字符 'c' 数量不足而不满足条件，我们收缩左边界，可能会移出一个出现次数很多的字符 'b'，反而让 'b' 的数量也不足了。这种条件的“非单调性”让常规滑动窗口难以施展。

面对这种困境，一个强大的思想是**增加约束，减少可变性**。

我们可以换个角度思考：一个合法的最长子串，它内部包含的**不同字符的种类数**（我们称之为 `h`）一定是一个确定的值，范围在 1 到 26 之间。

那我们能否**枚举**这个 `h` 呢？

如果我们固定了 `h`，问题就转化为：**找到最长的子串，它恰好包含 `h` 种不同的字符，且每种字符的出现次数都不少于 `k`。**

这个转化后的问题就变得非常清晰，并且完美符合滑动窗口的模式：
1.  我们寻找一个窗口，其**不同字符数最多为 `h`**。
2.  当不同字符数恰好为 `h` 时，我们再检查这 `h` 个字符是否都满足**出现次数不少于 `k`**。

这个思路将原问题分解为 26 个独立的、可以用滑动窗口解决的子问题。我们对每个子问题求最优解，最终取其中的最大值即可。

### 算法详解

1.  **外层循环**：我们用一个循环来枚举最终子串可能包含的唯一字符数 `h`，`for h in range(1, 27)`。

2.  **内层滑动窗口**：对于每一个固定的 `h`，我们使用滑动窗口 `[left, right]` 来寻找满足条件的子串。
    *   我们需要几个变量来维护窗口的状态：
        *   `counts`: 一个频率数组或哈希表，记录窗口内每个字符的出现次数。
        *   `unique_chars`: 记录当前窗口内不同字符的种类数。
        *   `chars_at_least_k`: 记录当前窗口内出现次数不少于 `k` 的字符种类数。
    *   **扩张窗口**：移动 `right` 指针，将新字符 `s[right]` 纳入窗口，并更新上述三个状态变量。
    *   **收缩窗口**：当 `unique_chars` 的值**超过**了我们当前枚举的 `h` 时，说明当前窗口太大了，需要从左侧收缩。我们移动 `left` 指针，将 `s[left]` 移出窗口，并反向更新三个状态变量，直到 `unique_chars` 重新等于 `h` 为止。
    *   **更新答案**：在窗口滑动的每一步，当它同时满足以下两个条件时，就意味着我们找到了一个候选的合法子串：
        1.  `unique_chars == h` (窗口内恰好有 `h` 种字符)
        2.  `chars_at_least_k == h` (这 `h` 种字符的出现次数都不少于 `k`)
        此时，我们用当前窗口的长度 `right - left + 1` 来更新全局的最大长度 `ans`。

3.  **返回结果**：外层循环结束后，`ans` 中保存的就是所有 `h` 值对应的最长子串中的最大值，即为最终答案。

### Python 代码实现

```python
class Solution:
    def longestSubstring(self, s: str, k: int) -> int:
        max_len = 0
        n = len(s)
        
        # 1. 外层循环：枚举窗口内允许的唯一字符数量 h
        # h 的取值范围是 [1, 26]，因为 s 仅由小写英文字母组成
        for h in range(1, 27):
            left = 0
            # 2. 内层滑动窗口
            # counts 用于记录窗口内各字符的出现次数
            counts = [0] * 26
            # unique_chars 记录窗口内唯一字符的数量
            unique_chars = 0
            # chars_at_least_k 记录窗口内满足出现次数 >= k 的字符数量
            chars_at_least_k = 0
            
            for right in range(n):
                # 窗口右边界扩张，新字符进入窗口
                char_right_idx = ord(s[right]) - ord('a')
                
                # 更新 counts, unique_chars, chars_at_least_k
                if counts[char_right_idx] == 0:
                    unique_chars += 1
                counts[char_right_idx] += 1
                if counts[char_right_idx] == k:
                    chars_at_least_k += 1
                
                # 3. 当窗口内唯一字符数 > h 时，收缩左边界
                # 这是为了维持窗口内最多只有 h 个唯一字符的约束
                while unique_chars > h:
                    char_left_idx = ord(s[left]) - ord('a')
                    
                    # 字符离开窗口，对应更新 counts, unique_chars, chars_at_least_k
                    if counts[char_left_idx] == k:
                        chars_at_least_k -= 1
                    counts[char_left_idx] -= 1
                    if counts[char_left_idx] == 0:
                        unique_chars -= 1
                    
                    left += 1
                
                # 4. 当窗口内唯一字符数等于 h，并且这些字符都满足 >= k 的条件时，
                # 我们找到了一个候选子串，更新最大长度
                if unique_chars == h and chars_at_least_k == h:
                    max_len = max(max_len, right - left + 1)
                    
        return max_len
```

### 复杂度分析

*   **时间复杂度**: O(N)。虽然代码有两层循环，但外层循环的次数是常数 26。内层循环是一个标准的滑动窗口，`left` 和 `right` 指针都只会从头到尾移动一次，所以其复杂度为 O(N)。因此总时间复杂度为 O(26 * N) = O(N)。
*   **空间复杂度**: O(1)。我们使用了一个大小为 26 的 `counts` 数组来存储字符频率，其空间占用是常数级别的。

### 总结

本题是滑动窗口思想的一个巧妙应用。当遇到一个性质不稳定的目标条件时，可以尝试通过**枚举其中一个可变维度**（本题中是唯一字符数）的方式，将其转化为一个具有稳定约束的子问题。这种“枚举约束 + 滑动窗口”的模式，是解决一系列复杂字符串和数组问题的有力武器。
