LeetCode 第 2616 题：[最小化数对的最大差值](https://leetcode.cn/problems/minimize-the-maximum-difference-of-pairs/description/)。
当直接求解“最小化的最大差值”很困难时，我们可以反向思考：给定一个差值 `d`，能否找到 `p` 个数对？这个问题可以通过二分查找将求解最值问题转化为判定问题。
{% endcenterquote %}
<!--more-->

### 问题解读

给定一个整数数组 `nums` 和一个整数 `p`。我们需要从数组中找出 `p` 个互不重叠的数对。我们的目标是，在这 `p` 个数对的差值中，找到那个最大的差值，并让这个“最大的差值”尽可能地小。最终返回这个“最小的可能的最大差值”。

举个例子，`nums = [10,1,2,7,1,3], p = 2`。
我们希望找出 2 个数对，使得它们差值的最大值最小。
为了让数对的差值尽可能小，我们应该优先选择数值上接近的数字配对。因此，一个关键的预处理步骤是对数组进行排序，得到 `[1,1,2,3,7,10]`。
排序后，相邻的元素之间的差值是最小的。
*   我们可以选择 `(1,1)`，差值为 0。
*   剩下 `[2,3,7,10]`，我们可以选择 `(2,3)`，差值为 1。
这样我们找到了两个数对 `(1,1)` 和 `(2,3)`，它们的差值分别是 0 和 1。这 2 个差值的最大值是 `max(0, 1) = 1`。这是一个可能的答案。我们能否找到更小的最大差值呢？答案是否定的，所以最终结果是 1。

### 核心思路：将“求解问题”转化为“判定问题”与二分查找

直接计算“最小的最大差值”没有明确的下手点。但是，我们可以转换思路：如果我们**猜测**一个“最大差值”为 `d`，我们能否判断出，是否存在 `p` 个数对，使得每个数对的差值都**不超过** `d`？

这个“判定问题”就清晰多了。我们可以定义一个 `check(d)` 函数：
*   该函数接收一个猜测的差值 `d` 作为参数。
*   在数组中，我们尝试寻找 `p` 个差值不大于 `d` 的数对。
*   为了尽可能多地凑出数对，我们应该采用**贪心策略**。首先对数组排序，然后遍历数组。当我们遇到 `nums[i+1] - nums[i] <= d` 时，就将 `(nums[i], nums[i+1])` 凑成一对。因为 `nums[i]` 与 `nums[i+1]` 的差值是最小的，如果它都不与 `nums[i+1]` 配对，那么它与后面的任何元素配对的差值只会更大，更不划算。同时，为了满足“每个下标最多出现一次”的规则，当我们成功凑对后，下一个检查点应该跳到 `i+2`。
*   如果在遍历结束后，我们能凑够 `p` 对，那么 `check(d)` 就返回 `True`。

这个 `check(d)` 函数具有非常明显的**单调性**：
*   如果一个最大差值 `d` 是**可行**的（能找到 `p` 对），那么任何比 `d` 更大的差值 `d'` 也**一定**是可行的（因为放宽了条件，原来满足的数对现在依然满足）。
*   反之，如果差值 `d` **不可行**（找不到 `p` 对），那么任何比 `d` 更小的差值 `d''` 也**必然**不可行。

这种单调性让我们能够立刻想到使用**二分查找**来解决问题。我们可以在所有可能的差值范围内对“答案”进行二分查找，从而快速定位到那个“最小的可行差值”。

这个二分查找的搜索范围是什么呢？
*   **下界 (left)**：`0`，差值最小可能为0。
*   **上界 (right)**：排序后数组的最大值与最小值之差，即 `nums[n-1] - nums[0]`。这是可能的最大差值。

因此，我们就在 `[0, nums[n-1] - nums[0]]` 这个区间内，对“最大差值”进行二分查找。

### 算法详解

1.  **排序数组**
    *   首先对 `nums` 数组进行升序排序。这是贪心策略的基础，使得相邻元素的差值最小。

2.  **确定二分查找的范围**
    *   `left = 0`：可能的最小答案。
    *   `right = nums[n-1] - nums[0]`：可能的最大答案。

3.  **执行二分查找**
    *   当 `left < right` 时，循环继续。
    *   计算中间值 `mid = left + (right - left) // 2` 作为当前的“猜测的最大差值”。
    *   调用辅助函数 `check(mid)` 来判定这个猜测值是否可行。
        *   `check(mid)` 函数内部，使用贪心策略进行检查：
            *   初始化一个计数器 `count = 0` 和一个指针 `i = 0`。
            *   当 `i < n-1` 时，检查 `nums[i+1] - nums[i]`。
            *   如果 `nums[i+1] - nums[i] <= mid`，说明找到一个合格的数对。我们让 `count` 加一，并且因为 `nums[i]` 和 `nums[i+1]` 都已被使用，指针 `i` 向后移动 2 位。
            *   如果 `nums[i+1] - nums[i] > mid`，说明 `(nums[i], nums[i+1])` 无法凑对。我们只能跳过 `nums[i]`，让指针 `i` 向后移动 1 位，继续寻找机会。
            *   最终，如果 `count >= p`，则 `check(mid)` 返回 `True`。
        *   **如果 `check(mid)` 返回 `True`** (即最大差值为 `mid` 时，可以凑够 `p` 对)：
            *   说明 `mid` 是一个潜在的答案，但我们想找的是“最小”的最大差值，所以我们尝试一个更小的值，将搜索范围缩小到左半边（包含 mid）：`right = mid`。
        *   **如果 `check(mid)` 返回 `False`** (即最大差值为 `mid` 时，凑不够 `p` 对)：
            *   说明我们对最大差值的限制太严格了，需要放宽一些，将搜索范围缩小到右半边：`left = mid + 1`。

4.  **返回结果**
    *   循环结束后，`left` 和 `right` 相遇，此时的值就是满足条件的最小的最大差值。

### Python 代码实现

```python
from typing import List

class Solution:
    def minimizeMax(self, nums: List[int], p: int) -> int:
        # 对数组排序，这是贪心策略的基础
        nums.sort()
        n = len(nums)

        # check(mx) 函数用于判定：
        # 当允许的最大差值为 mx时，能否找到 p 个数对
        def check(mx: int) -> bool:
            cnt = 0  # 记录找到的有效数对数量
            i = 0
            while i < n - 1:
                # 如果相邻元素的差值小于等于 mx，则它们可以组成一个数对
                if nums[i+1] - nums[i] <= mx:
                    cnt += 1
                    # 这两个元素已被使用，跳过它们
                    i += 2
                else:
                    # 无法配对，只跳过当前元素
                    i += 1
            # 如果找到的数对数量大于等于 p，则说明 mx 是一个可行的最大差值
            return cnt >= p

        # 二分查找的左右边界
        # 左边界：最小可能的差值
        # 右边界：最大可能的差值
        l, r = 0, nums[-1] - nums[0]
        
        # 二分查找寻找最小的可行 mx
        while l < r:
            mid = (l + r) // 2
            if check(mid):
                # 如果 mid 可行，说明它可能是答案，
                # 或者还有更小的可行值，所以尝试在左半部分 [l, mid] 继续寻找
                r = mid
            else:
                # 如果 mid 不可行，说明最大差值限制太小，
                # 必须放宽，所以去右半部分 [mid + 1, r] 寻找
                l = mid + 1
        
        # 循环结束时，l (或 r) 就是满足条件的最小的最大差值
        return r
```

### 复杂度分析

*   **时间复杂度**: O(N * log(Range) + N * logN)，其中 N 是数组的长度，Range 是数组最大值与最小值的差。
    *   对数组排序需要 O(N * logN) 的时间。
    *   二分查找的搜索空间大小为 `Range`，因此迭代次数为 O(log(Range))。
    *   在每次二分迭代中，我们都需要调用 `check` 函数，`check` 函数需要对数组进行一次线性扫描，时间复杂度为 O(N)。
    *   因此，总时间复杂度为 O(N * logN + N * log(Range))。
*   **空间复杂度**: O(logN) 或 O(N)。
    *   空间开销主要来自于排序算法。如果使用基于快速排序的内省排序，空间复杂度为 O(logN)；如果使用归并排序，则为 O(N)。除了排序之外，算法本身只使用了常数级别的额外空间。

### 总结

本题是“最小化最大值”问题的一个绝佳例子。它完美地诠释了如何通过二分查找答案的思路，将一个复杂的求解最优化问题，转换成一个更简单的、具有单调性的判定性问题。当遇到类似“求...的最小值”或“求...的最大值”，且难以直接求解时，不妨思考一下是否能对答案进行二分，设计一个 `check` 函数来验证猜测值的可行性。如果可行，这种方法往往能提供一个非常高效且优雅的解决方案。
