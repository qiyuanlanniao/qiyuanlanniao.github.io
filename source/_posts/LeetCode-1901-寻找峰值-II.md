LeetCode 第 1901 题：[寻找峰值 II](https://leetcode.cn/problems/find-a-peak-element-ii/)。
题目要求时间复杂度为 O(m log n) 或 O(n log m)，这提示我们需要在一个维度上进行二分查找，在另一个维度上进行线性扫描。
{% endcenterquote %}
<!--more-->

### 问题描述

一个 2D 网格中的 **峰值** 是指那些 **严格大于** 其相邻格子(上、下、左、右)的元素。

给你一个 从 0 开始编号 的 `m x n` 矩阵 `mat` ，其中任意两个相邻格子的值都 **不相同** 。找出 **任意一个** 峰值 `mat[i][j]` 并 返回其位置 `[i,j]` 。

你可以假设整个矩阵周边环绕着一圈值为 -1 的格子。

**示例 1:**
```text
输入: mat = [[1,4],[3,2]]
输出: [0,1]
解释: 3 和 4 都是峰值，所以[1,0]和[0,1]都是可接受的答案。
```

**示例 2:**
```text
输入: mat = [[10,20,15],[21,30,14],[7,16,32]]
输出: [1,1]
解释: 30 和 32 都是峰值，所以[1,1]和[2,2]都是可接受的答案。
```

**提示：**
*   m == mat.length
*   n == mat[i].length
*   1 <= m, n <= 500
*   1 <= mat[i][j] <= 10^5
*   任意两个相邻元素均不相等

### 解题思路

#### 1. 为什么不能暴力遍历？

如果直接遍历整个矩阵寻找峰值，时间复杂度是 O(m * n)。虽然对于 500x500 的数据规模来说可以通过，但题目明确要求 O(m log n) 或 O(n log m) 的复杂度，这意味着我们必须利用二分查找的思想。

#### 2. 行二分查找策略

我们可以对 **行** 进行二分查找，具体步骤如下：

1.  **二分行号**：在行索引 `0` 到 `m-1` 之间进行二分，设当前中间行为 `i`。
2.  **寻找行最大值**：在第 `i` 行中找到最大值 `mat[i][j]`。
    *   为什么找行最大值？因为如果一个位置是该行的最大值，那么它一定严格大于它的左边和右边（如果存在）。此时要判断它是不是峰值，只需要看它的 **上** 和 **下** 两个邻居。
3.  **比较相邻行**：将 `mat[i][j]` 与它下方的元素 `mat[i+1][j]` 进行比较：
    *   **情况 A**：`mat[i][j] > mat[i+1][j]`。
        说明从第 `i` 行往下走是“下坡”。根据爬坡理论，如果我们往数值大的方向走一定能找到峰值。既然下方变小了，说明峰值不可能在下方（或者说上方一定存在一个峰值，可能是 `mat[i][j]` 自己，也可能是更上面的某处）。因此，我们可以舍弃 `i` 下面的所有行，将搜索范围缩小到上这半部分（包含第 `i` 行）。
    *   **情况 B**：`mat[i][j] < mat[i+1][j]`。
        说明从第 `i` 行往下走是“上坡”。这意味着第 `i` 行肯定不是峰值所在行（因为有个邻居比它大），且下方一定存在一个峰值。因此，我们将搜索范围缩小到 `i` 的下方。

#### 3. 算法复杂度

*   **二分过程**：我们对行号 `0` 到 `m-1` 进行二分，次数为 O(log m)。
*   **查找最大值**：每次二分内部，我们需要遍历当前行的 `n` 个元素找到最大值，耗时 O(n)。
*   **总复杂度**：O(n log m)。这完全符合题目要求。

```python
class Solution:
    def findPeakGrid(self, mat: List[List[int]]) -> List[int]:
        # 二分查找的上下界
        # 采用开区间写法 (l, r)，l 和 r 分别是这一轮不可能的边界和可能的边界
        # 初始 l = -1 (假想的上边界), r = len(mat) - 1 (最后一行索引)
        l, r = -1, len(mat) - 1
        
        # 当 l + 1 < r 时，说明区间内至少还有一个待检查的元素
        while l + 1 < r:
            # 取中间行
            i = (l + r) // 2
            
            # 找到第 i 行的最大值
            # 时间复杂度 O(n)
            mx = max(mat[i])
            
            # 找到最大值所在的列索引
            j = mat[i].index(mx)
            
            # 比较当前行最大值与下一行对应位置的值
            # 如果当前行最大值 > 下一行对应值，说明峰值在当前行或上方
            # 注意：i 是二分的中点，由于 l+1 < r，i 至少比 r 小 1，
            # 只有当 len(mat) > 1 时才会进入循环，此时 i+1 不会越界
            # 实际上若只有一行，循环不执行，直接返回第0行结果
            if mx > mat[i+1][j]:
                r = i  # 峰值行号 <= i，缩小右边界
            else:
                l = i  # 峰值行号 > i，缩小左边界（mat[i][j] < mat[i+1][j]）
                
        # 循环结束后，r 指向满足条件的行
        # 重新找到该行的最大值位置并返回
        return [r, mat[r].index(max(mat[r]))]
```

```go
func findPeakGrid(mat [][]int) []int {
    // 初始化二分查找的上下边界
    // left 为开区间左边界（不包含），right 为闭区间右边界（包含）
    left, right := -1, len(mat)-1
    
    // 开区间二分查找模板
    for left+1 < right {
        // 计算中间行索引
        i := left + (right-left)/2
        
        // 找到第 i 行最大值的列索引 j
        j := indexOfMax(mat[i])
        
        // 比较当前值与它下方邻居的值
        // 注意：由于是向下取整且 left+1 < right，i+1 不会越界
        if mat[i][j] > mat[i+1][j] {
            right = i // 当前值更大，说明峰值可能在第 i 行或其上方
        } else {
            left = i // 下方值更大，说明第 i 行不是峰值行，峰值在下方
        }
    }
    
    // right 最终指向峰值所在的行
    return []int{right, indexOfMax(mat[right])}
}

    for i, x := range a {
        if x > a[idx] {
            idx = i
        }
    }

}
```

### 复杂度分析

*   **时间复杂度**: `O(n log m)`
    我们需要对 `m` 行进行二分查找，这需要 `log m` 次迭代。在每一次迭代中，我们需要遍历一行 `n` 个元素来找到最大值。因此总的时间复杂度是 `n * log m`。
    
*   **空间复杂度**: `O(1)`
    我们只使用了常数个变量（如 `left`, `right`, `i`, `mx` 等）来存储状态，不需要额外的存储空间（除了输入矩阵本身）。
