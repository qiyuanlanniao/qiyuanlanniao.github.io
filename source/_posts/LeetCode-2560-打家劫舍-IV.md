LeetCode 第 2560 题：[打家劫舍 IV](https://leetcode.cn/problems/house-robber-iv/description/)。
本题的核心在于识别出答案的单调性，并将一个求解最优值（最小的最大值）的问题，巧妙地转化为一个判定性的问题，从而利用二分查找在对数时间内高效求解。
{% endcenterquote %}
<!--more-->

### 问题解读

我们需要在一排房屋中窃取至少 `k` 间，但不能窃取相邻的房屋。小偷的“窃取能力”被定义为他在一次成功的窃取行动中，从单间房屋里拿到的**最大**金额。我们的目标是，在所有满足条件的窃取方案中，找到这个“窃取能力”的**最小值**。

这里的核心约束条件是：
1.  不能窃取相邻的房屋。
2.  总共窃取的房屋数量必须**至少**为 `k` 间。
3.  目标是让窃取方案中，金额最大的那间房屋的金额尽可能小。

举个例子，`nums = [2,3,5,9]`, `k = 2`。
*   方案一：窃取 2 和 5。窃取能力是 `max(2, 5) = 5`。
*   方案二：窃取 2 和 9。窃取能力是 `max(2, 9) = 9`。
*   方案三：窃取 3 和 9。窃取能力是 `max(3, 9) = 9`。
在所有可行的方案中，最小的窃取能力是 5。

这个问题本质上是在求解“最小值中的最大值”或者“最大值中的最小值”这类极小化极大问题，这通常是二分答案的经典应用场景。

### 核心思路：将“求解问题”转化为“判定问题”与二分查找

直接去枚举所有组合来求解“最小的窃取能力”是非常复杂的。因此，我们不妨换一个角度思考：如果我们**假定**一个窃取能力为 `c`，我们能否判断出，是否存在一个合法的方案，使得我们能偷到至少 `k` 间房，并且所有被偷的房屋金额都不超过 `c`？

这个判定问题就相对简单了。我们可以设计一个辅助函数 `check(capacity)`，用贪心的策略来验证：
*   我们遍历所有房屋。
*   对于第 `i` 间房，如果它的金额 `nums[i]` 不超过我们假定的能力 `capacity`，那么我们就应该窃取它，因为这能帮助我们尽快凑够 `k` 间房。
*   一旦我们决定窃取第 `i` 间房，根据规则，我们必须跳过第 `i+1` 间房。
*   如果我们不窃取第 `i` 间房（因为它金额太高），那我们就继续考察第 `i+1` 间房。
*   遍历结束后，我们统计总共窃取了多少间房。如果数量大于等于 `k`，说明 `capacity` 这个能力是可行的。

现在，我们有了一个判定的方法。这个判定结果具有明显的**单调性**：
*   如果一个能力 `c` 可以让我们偷到至少 `k` 间房，那么任何比 `c` 更大的能力 `c'` 也一定可以（因为更多的房屋金额会达标，我们只会有更多选择）。
*   反之，如果 `c` 无法满足条件，那么任何比 `c` 更小的能力也必然无法满足。

这种单调性正是应用二分查找的完美前提。我们可以对“窃取能力”这个值进行二分查找，来快速定位到那个“最小可行”的临界点。

这个二分查找的搜索范围是什么呢？
*   **下界 (left)**：窃取能力至少得是房屋金额中的最小值，否则一间都偷不了。所以下界是 `min(nums)`。
*   **上界 (right)**：窃取能力最大也只需要是房屋金额中的最大值，因为更大的能力不会改变任何决策。所以上界是 `max(nums)`。

因此，我们就在 `[min(nums), max(nums)]` 这个区间内，对窃取能力进行二分查找。

### 算法详解

1.  **确定二分查找的范围**
    *   `left = min(nums)`：这是可能的最低能力。
    *   `right = max(nums)`：这是可能的最高能力。

2.  **执行二分查找**
    *   当 `left < right` 时，循环继续。
    *   计算中间值 `mid = left + (right - left) // 2` 作为当前的“猜测能力”。
    *   调用辅助函数 `check(mid)` 来判定这个猜测值是否可行。
        *   **如果 `check(mid)` 返回 `True`** (即 `mid` 这个能力可以偷到至少 `k` 间房)：
            *   说明 `mid` 是一个潜在的答案，但我们想找的是“最小”能力，所以我们尝试一个更小的值。我们将搜索范围缩小到左半边（包含 `mid`）：`right = mid`。
        *   **如果 `check(mid)` 返回 `False`** (即 `mid` 这个能力不够)：
            *   说明我们需要更大的能力，将搜索范围缩小到右半边：`left = mid + 1`。

3.  **返回结果**
    *   循环结束后，`left` 和 `right` 会相遇于一点，这个点就是满足条件的最小窃取能力。

### Python 代码实现

```python
from typing import List

class Solution:
    def minCapability(self, nums: List[int], k: int) -> int:
        
        # 确定二分查找的左右边界
        left, right = min(nums), max(nums)
        
        # 辅助函数：检查在给定的窃取能力(capacity)下，能否窃取至少 k 间房屋
        def check(capacity: int) -> bool:
            count = 0  # 记录可以窃取的房屋数量
            i = 0
            n = len(nums)
            while i < n:
                if nums[i] <= capacity:
                    # 这间房的金额在能力范围内，可以偷
                    # 偷了这间，下一间必须跳过
                    count += 1
                    i += 2
                else:
                    # 金额太高，不能偷，看下一间
                    i += 1
            return count >= k

        # 二分查找
        while left < right:
            mid = left + (right - left) // 2
            if check(mid):
                # 如果 mid 可行，它可能就是答案，
                # 或者还有更小的可行值，所以我们尝试在左半部分继续寻找
                right = mid
            else:
                # 如果 mid 不可行，说明能力太小，
                # 必须增大，所以去右半部分寻找
                left = mid + 1
        
        # 循环结束时，left 就是满足条件的最低窃取能力
        return left
```

### 复杂度分析

*   **时间复杂度**: O(N * log(M))。
    *   `N` 是房屋数组 `nums` 的长度。
    *   `M` 是 `max(nums) - min(nums)`，即二分查找的搜索空间大小。
    *   二分查找本身需要 `log(M)` 次迭代。
    *   在每次迭代中，我们都需要调用 `check` 函数，该函数需要贪心地遍历一次 `nums` 数组，时间复杂度为 O(N)。
    *   因此，总时间复杂度是这两者的乘积。
*   **空间复杂度**: O(1)。
    *   除了输入数组外，我们只使用了几个变量来存储左右边界和中间值，所以空间复杂度是常数级的。

### 总结

本题是“二分答案”思想的又一个绝佳应用。它向我们展示了如何将一个看似复杂的“求最优解”问题，转化为一个结构简单且具有单调性的“判定”问题。一旦建立起这种转化思维，就可以利用二分查找这一强大工具，将问题的时间复杂度从指数级或高次多项式级，优化到对数级别，从而高效地解决问题。
