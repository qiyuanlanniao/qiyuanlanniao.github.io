LeetCode 第 378 题：[有序矩阵中第 K 小的元素](https://leetcode.cn/problems/kth-smallest-element-in-a-sorted-matrix/description/)。
对于在一个值域范围内查找“第 K 小”或“第 K 大”的问题，尤其是当数据本身具有一定的有序性（如行、列分别排序）时，**二分答案** 往往是优于排序和堆的更优解法。本题利用矩阵的性质，将时间复杂度从多项式级降低到了对数级。
{% endcenterquote %}
<!--more-->

### 题目描述

给你一个矩阵 `matrix` ，其中每行和每列元素均按升序排序，找到矩阵中第 `k` 小的元素。
请注意，它是 **排序后** 的第 `k` 小元素，而不是第 `k` 个 **不同** 的元素。

你必须找到一个内存复杂度优于 $O(n^2)$ 的解决方案。

**示例 1：**
> **输入：** matrix = [[1,5,9],[10,11,13],[12,13,15]], k = 8
> **输出：** 13
> **解释：** 矩阵中的元素为 [1,5,9,10,11,12,13,13,15]，第 8 小元素是 13

**示例 2：**
> **输入：** matrix = [[-5]], k = 1
> **输出：** -5

### 解题思路

#### 二分查找（二分答案）

我们注意到答案一定在 `matrix[0][0]`（最小值）和 `matrix[n-1][n-1]`（最大值）之间。
这不仅是一个查找问题，更是一个**值域上的判定问题**：

> 猜测一个整数 `x`，如果矩阵中 **小于等于 `x` 的元素个数** 大于等于 `k`，说明第 `k` 小的元素一定在 `x` 左边（或者就是 `x`）；否则，答案一定在 `x` 右边。

具有这种**单调性**（随着 `x` 增大，小于等于 `x` 的元素个数单调递增），我们就可以使用**二分查找**。

##### 核心步骤：

1.  **确定二分范围**：
    *   左边界 `l`：`matrix[0][0]`
    *   右边界 `r`：`matrix[n-1][n-1]`
2.  **Check 函数的设计**：
    我们需要一个快速的方法计算矩阵中有多少个元素 $\le mid$。
    利用矩阵 **行增列增** 的性质，我们可以从矩阵的 **右上角**（或左下角）开始遍历：
    *   初始位置 $(i, j) = (0, n-1)$。
    *   如果当前元素 `matrix[i][j] > mid`：说明这一列剩下的元素都比 `mid` 大，排除这一列，`j--`。
    *   如果当前元素 `matrix[i][j] <= mid`：说明这一行从起点到 `j` 的元素都符合条件（共 `j + 1` 个），累加计数，并向下移动 `i++` 寻找更多符合条件的元素。
    *   该过程的时间复杂度为 $O(n)$，因为 $i$ 和 $j$ 最多各移动 $n$ 次。
3.  **二分逻辑**：
    *   使用开区间或半开区间的二分模版来逼近答案。
    *   如果 `check(mid)` 返回 `True`（计数 $\ge k$），说明 `mid` 可能是答案，或者是比答案大的数，尝试缩小右边界。
    
### 代码实现

```python
class Solution:
    def kthSmallest(self, matrix: List[List[int]], k: int) -> int:
        n = len(matrix)
        
        # check 函数：统计矩阵中 <= mx 的元素个数是否 >= k
        # 时间复杂度：O(n)，利用了矩阵行列递增的性质
        def check(mx: int) -> bool:
            cnt = 0
            i, j = 0, n - 1
            # 从右上角开始走
            while i < n and j >= 0 and cnt < k:
                if matrix[i][j] > mx:
                    # 当前元素大于 mx，这一列下面的元素肯定也大于 mx，向左移
                    j -= 1
                else:
                    # 当前元素 <= mx，说明这一行左边所有元素（包括自己）都 <= mx
                    # 个数为 j + 1 (下标从0开始)
                    cnt += j + 1
                    # 继续看下一行
                    i += 1
            return cnt >= k
        
        # 二分查找的值域范围
        # 使用开区间 (l, r] 的逻辑，l 初始为最小值-1
        l, r = matrix[0][0] - 1, matrix[-1][-1]
        
        while l + 1 < r:
            mid = (l + r) // 2
            if check(mid):
                # 如果小于等于 mid 的数量够 k 个，说明答案在 [l+1, mid] 范围内
                # 尝试更小的值
                r = mid
            else:
                # 数量不够，说明 mid 太小了，答案在 [mid+1, r]
                l = mid
                
        return r
```
