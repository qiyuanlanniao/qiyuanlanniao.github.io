LeetCode 第 719 题：[找出第 K 小的数对距离](https://leetcode.cn/problems/find-k-th-smallest-pair-distance/description/)。
本题是经典的“第 K 小”问题，解题核心在于将求解转化为判定：给定一个距离 x，统计距离 ≤x 的数对个数 cnt，判断是否满足 cnt≥k。结合排序和双指针技巧，可以将判定过程优化至线性时间。
{% endcenterquote %}
<!--more-->

### 问题描述

数对 `(a,b)` 由整数 `a` 和 `b` 组成，其数对距离定义为 `a` 和 `b` 的绝对差值。

给你一个整数数组 `nums` 和一个整数 `k` ，数对由 `nums[i]` 和 `nums[j]` 组成且满足 `0 <= i < j < nums.length` 。返回 所有数对距离中 第 `k` 小的数对距离。

**示例 1：**

```
输入：nums = [1,3,1], k = 1
输出：0
解释：数对和对应的距离如下：
(1,3) -> 2
(1,1) -> 0
(3,1) -> 2
距离第 1 小的数对是 (1,1) ，距离为 0 。
```

**示例 2：**

```
输入：nums = [1,1,1], k = 2
输出：0
```

**示例 3：**

```
输入：nums = [1,6,1], k = 3
输出：5
```

**提示：**
*   `n == nums.length`
*   `2 <= n <= 10^4`
*   `0 <= nums[i] <= 10^6`
*   `1 <= k <= n * (n - 1) / 2`

### 解题思路

#### 暴力解法与瓶颈

最直观的方法是计算出所有可能的数对距离，放入一个数组中，然后进行排序并取第 `k` 个值。
数组长度为 `N`，则总共有 `N * (N - 1) / 2` 个数对。
当 `N = 10^4` 时，数对数量约为 `5 * 10^7`。对这么多数据进行排序，时间复杂度为 `O(N^2 * log(N^2))`，这显然会**超时**（TLE），甚至可能导致**内存溢出**（MLE）。

#### 二分答案

既然无法直接计算并排序所有距离，我们需要换个角度。
数对距离的取值范围是确定的：
*   最小值：`0`（当数组中有重复元素时）。
*   最大值：数组中最大元素与最小元素的差值。

我们可以在这个**数值范围** `[0, max(nums) - min(nums)]` 内进行二分查找。
假设我们猜测第 `k` 小的距离是 `mid`。为了验证，我们需要计算：**数组中有多少个数对的距离小于等于 `mid`？** 记这个数量为 `cnt`。

*   如果 `cnt >= k`：说明小于等于 `mid` 的数对太多了，或者正好足够。第 `k` 小的距离可能是 `mid`，也可能比 `mid` 更小。答案在左半区间（包含 `mid`）。
*   如果 `cnt < k`：说明 `mid` 太小了，连前 `k` 个都覆盖不到。答案一定比 `mid` 大。答案在右半区间。

#### 使用双指针（滑动窗口）优化计数

现在的核心难点在于：如何高效地计算“距离小于等于 `mid` 的数对个数”？
如果在未排序的数组中统计，仍需 `O(N^2)` 的时间，二分也就失去了意义。
但是，如果我们**先对数组进行排序**，就可以利用单调性使用**双指针**来统计。

假设数组已排序，对于每一个右端点 `j`，我们要找到最小的左端点 `i`，使得 `nums[j] - nums[i] <= mid`。
由于数组是递增的，随着 `j` 向右移动（变大），满足条件的 `i` 也只会向右移动（或不动），而不会回头。

算法流程：
1.  对 `nums` 排序。
2.  枚举右指针 `j` 从 `0` 到 `n-1`。
3.  当 `nums[j] - nums[i] > mid` 时，说明当前的 `i` 太小了（导致差值太大），需要右移 `i`。
4.  对于当前的 `j`，有效数对的数量为 `j - i`（即索引 `i` 到 `j-1` 都可以和 `j` 组成满足条件的数对）。
5.  累加所有 `j` 对应的数量。

这个 check 函数的时间复杂度是 `O(N)`。

### 代码实现

代码中使用了“开区间”二分模板（`l` 始终不满足条件，`r` 始终满足条件），这是一种不易写错死循环的写法。

```python
class Solution:
    def smallestDistancePair(self, nums: List[int], k: int) -> int:
        # 1. 排序，这是使用双指针的前提
        nums.sort()

        # check 函数：统计有多少个数对的距离 <= mx
        def check(mx: int) -> bool:
            cnt = 0
            i = 0
            # 双指针：j 是右指针，i 是左指针
            for j, x in enumerate(nums):
                # 如果当前距离大于 mx，说明 i 太靠左了，也就是 nums[i] 太小
                # 此时需要移动左指针 i，直到满足条件
                while x - nums[i] > mx:
                    i += 1
                # 此时 [i, j-1] 范围内的数与 nums[j] 的距离都 <= mx
                # 个数为 j - i
                cnt += j - i
            return cnt >= k

        # 2. 二分查找
        # 距离的可能范围是 [0, nums[-1] - nums[0]]
        # 这里使用开区间写法 (l, r]
        # l 初始为 -1 (不可能的极小值)
        # r 初始为最大距离 (必然满足条件的极大值)
        l, r = -1, nums[-1] - nums[0]
        
        while l + 1 < r:
            mid = (l + r) // 2
            if check(mid):
                # 如果 count >= k，说明 mid 可能是答案，或者答案更小
                # mid 是满足条件的一个候选值，收缩右边界
                r = mid
            else:
                # count < k，说明 mid 太小了，答案一定大于 mid
                # 收缩左边界
                l = mid
        
        return r
```

### 复杂度分析

*   **时间复杂度**: `O(N * log(N) + N * log(D))`。
    *   排序需要 `O(N * log(N))`。
    *   二分查找的范围 `D = max(nums) - min(nums)`，最大为 `10^6`，二分次数为 `log(D)`。
    *   每次二分调用 `check` 函数，双指针遍历数组需要 `O(N)`。
    *   总复杂度为 `O(N * log(N) + N * log(max(nums)))`。
*   **空间复杂度**: `O(1)` 或 `O(log(N))`。
    *   取决于排序算法的实现，双指针和二分查找仅使用常数级额外空间。
