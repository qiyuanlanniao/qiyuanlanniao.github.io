LeetCode 第 154 题：[寻找旋转排序数组中的最小值 II](https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array-ii/)。
该题难点在于数组中可能存在 **重复元素**。这破坏了二分查找原本严格的单调性，使得我们需要特殊处理相等的情况。
{% endcenterquote %}
<!--more-->

### 问题描述

已知一个长度为 `n` 的数组，预先按照升序排列，经由 1 到 `n` 次 **旋转** 后，得到输入数组。
例如，原数组 `nums = [0,1,4,4,5,6,7]` 在变化后可能得到：
*   若旋转 4 次，则可以得到 `[4,5,6,7,0,1,4]`
*   若旋转 7 次，则可以得到 `[0,1,4,4,5,6,7]`

给你一个可能存在 **重复** 元素值的数组 `nums`，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 **最小元素**。

你必须尽可能减少整个过程的操作步骤。

**示例 1：**
```text
输入：nums = [1,3,5]
输出：1
```

**示例 2：**
```text
输入：nums = [2,2,2,0,1]
输出：0
```

**提示：**
*   `n == nums.length`
*   `1 <= n <= 5000`
*   `-5000 <= nums[i] <= 5000`
*   `nums`原来是一个升序排序的数组，并进行了 1 至 n 次旋转

### 解题思路

#### 1. 旋转数组的性质

旋转排序数组（Rotated Sorted Array）通常可以看作是被分割成了两个有序的子数组。例如 `[4,5,6,7,0,1,2]`，左半部分 `[4,5,6,7]` 和右半部分 `[0,1,2]` 都是有序的，且右半部分的元素通常小于等于左半部分的元素（除非未旋转）。最小值就是右半部分的第一个元素。

#### 2. 二分查找策略

我们的目标是找到旋转点（最小值）。在标准的二分查找中（没有重复元素），我们通常比较 `nums[mid]` 和 `nums[right]`：

*   如果 `nums[mid] < nums[right]`：说明 `mid` 到 `right` 这部分是有序的，最小值肯定在 `mid` 左边或者就是 `mid` 自己。因此区间缩小为 `[left, mid]`。
*   如果 `nums[mid] > nums[right]`：说明 `mid` 处于左半段较大的有序部分，最小值一定在 `mid` 的右边。因此区间缩小为 `[mid + 1, right]`。

#### 3. 处理重复元素

本题的难点在于 **重复元素**，即 `nums[mid] == nums[right]` 的情况。
例如：
*   情况 A: `[1, 0, 1, 1, 1]`，`mid` 为 2，`nums[mid] = 1`，`nums[right] = 1`。最小值 0 在左边。
*   情况 B: `[1, 1, 1, 0, 1]`，`mid` 为 2，`nums[mid] = 1`，`nums[right] = 1`。最小值 0 在右边。

当 `nums[mid] == nums[right]` 时，我们无法判断最小值是在 `mid` 的左边还是右边。
但是，由于 `nums[mid]` 和 `nums[right]` 相等，我们可以确定的是：**哪怕我们忽略掉 `nums[right]` 这个位置，最小值的候选人 `nums[mid]` 依然保留在搜索范围内**。
所以，安全的做法是将右边界 `right` 向左移动一位（`right--`），逐步缩小范围，直到打破相等的状态。

#### 4. 开区间二分法

在提供的代码中，使用的是 `(l, r)` 开区间（或者说 `l` 是非解集边界，`r` 是解集边界）的写法：
*   `l` 初始化为 `-1`。
*   `r` 初始化为 `len(nums) - 1`。
*   循环条件 `l + 1 < r`。
*   最终返回 `nums[r]`。

这种写法下，`r` 始终指向“可能是最小值”的位置，而 `l` 指向“必定不是最小值”或“旋转点左侧的大值部分”的位置。

### 代码实现

<!-- tab Python -->
```python
class Solution:
    def findMin(self, nums: List[int]) -> int:
        # 初始化左右指针
        # 采用开区间写法，l 初始为 -1，r 初始为最后一个元素下标
        l, r = -1, len(nums) - 1
        
        # 当 l 和 r 相邻时停止循环，此时 r 指向最小值
        while l + 1 < r:
            mid = (r + l) // 2
            
            # 情况 1: 中间值等于右边界值
            # 无法确定最小值在左还是右，但由于 nums[mid] == nums[r]，
            # 我们可以安全地排除 nums[r]，因为即使它是最小值，nums[mid] 还在范围内
            if nums[mid] == nums[r]:
                r -= 1
            
            # 情况 2: 中间值小于右边界值
            # 说明 mid 到 r 区间是有序的（未旋转或旋转点在 mid 左侧）
            # 最小值可能是 mid 自己，或者是 mid 左边的元素
            # 收缩右边界到 mid
            elif nums[mid] < nums[r]:
                r = mid
            
            # 情况 3: 中间值大于右边界值
            # 说明 mid 在左半段较大的有序区间内
            # 最小值一定在 mid 的右侧
            # 收缩左边界到 mid
            else:
                l = mid
                
        # 循环结束时，r 指向最小值的位置
        return nums[r]
```

<!-- tab Go -->
```go
func findMin(nums []int) int {
    // 初始化左右指针
    // l 表示左边界（不包含），r 表示右边界（包含）
    l, r := -1, len(nums)-1
    
    // 保持 l 和 r 之间至少有一个元素
    for l+1 < r {
        mid := l + (r-l)/2
        
        // 情况 1: 遇到重复元素
        // 无法判断最小值方位，只能线性缩减右边界
        // 这是一个安全操作，因为 nums[mid] 和 nums[r] 值相同
        if nums[mid] == nums[r] {
            r--
        } else if nums[mid] < nums[r] {
            // 情况 2: 右半部分有序
            // 最小值在 [l+1, mid] 范围内（包括 mid）
            r = mid
        } else {
            // 情况 3: nums[mid] > nums[r]
            // 左半部分有序且值较大，最小值肯定在右边
            // 最小值在 [mid+1, r] 范围内
            l = mid
        }
    }
    // r 最终停留在最小值的索引上
    return nums[r]
}
```

### 复杂度分析

*   **时间复杂度**:
    *   **平均情况**: `O(log n)`。当数组中重复元素较少时，主要逻辑类似于二分查找。
    *   **最坏情况**: `O(n)`。当数组中包含大量重复元素（例如 `[1, 1, 1, 1]`）时，每次循环只能将 `r` 减少 1，退化为线性查找。
*   **空间复杂度**: `O(1)`。只使用了常数个变量。
