LeetCode 第 2439 题：[最小化数组中的最大值](https://leetcode.cn/problems/minimize-maximum-of-array/description/)。
本题的核心在于识别出答案的单调性，并将一个求解最优值（最小化最大值）的问题，巧妙地转化为一个判定性的问题，从而利用二分查找在对数时间内高效求解。
{% endcenterquote %}
<!--more-->

### 问题解读

我们得到一个非负整数数组 `nums`。题目定义了一种操作：
*   选择一个索引 `i`（`1 <= i < n`），且 `nums[i]` 必须大于 0。
*   将 `nums[i]` 的值减 1。
*   将 `nums[i - 1]` 的值加 1。

这个操作本质上是把 `nums[i]` 的一部分值“传递”给它左边的邻居 `nums[i-1]`。这个过程可以执行任意次。我们的目标是，通过这些操作，使得数组中最大的那个元素的值尽可能小。我们需要返回这个“最小的最大值”。

举个例子，`nums = [3,7,1,6]`。
我们可以执行一系列操作，比如把 `nums[1]` 的 2 挪到 `nums[0]`，把 `nums[3]` 的 1 挪到 `nums[2]`，最终可以得到 `[5,5,2,5]`。此时，数组中的最大值是 5。题目告诉我们，无法得到比 5 更小的最大值，所以 5 就是答案。

这个操作有一个非常重要的特性：**值只能向左移动**。`nums[i]` 的值可以影响 `nums[i-1], nums[i-2], ... , nums[0]`，但永远无法影响到它右边的 `nums[i+1]`。这暗示了数组的前缀和（prefix sum）可能是一个关键的不变量或约束条件。

### 核心思路：将“求解问题”转化为“判定问题”与二分查找

直接求解“最小的最大值”是复杂的。但我们可以换一个角度思考：如果我们**假设**一个最终的最大值 `m`，我们能否判断出这个目标是否可以实现？

这个问题（我们称之为“判定问题”）就清晰多了。我们可以编写一个辅助函数 `check(m)`，来判断是否能通过操作，使得数组中所有元素都小于等于 `m`。

如果我们可以高效地实现 `check(m)`，就会发现这个判定结果具有**单调性**：
*   如果一个值 `m` 是可行的（即我们可以让数组的最大值不超过 `m`），那么任何大于 `m` 的值 `m'` 也一定可行。
*   反之，如果 `m` 不可行，那么任何小于 `m` 的值也一定不可行。

这种单调性是应用二分查找的完美信号。我们可以对“可能的答案”进行二分查找，从而快速定位到那个“最小的可行值”。

那么，这个二分查找的搜索范围是什么呢？
*   **下界 (left)**：由于数组元素非负，最小的最大值不可能是负数，所以下界可以是 `0`。
*   **上界 (right)**：在最坏的情况下，我们不执行任何操作，此时最大值是原数组中的最大值 `max(nums)`。一个更宽松的上界可以是题目给定的 `10^9`。

因此，我们就在 `[0, max(nums)]` 这个区间内，对可能的答案进行二分查找。

### 算法详解

现在，核心落在了如何实现 `check(m)` 函数上。

回顾操作的特性：值只能从 `nums[i]` 移动到 `nums[i-1]`。这意味着，对于数组的任意一个前缀 `nums[0...i]`，这个前缀中所有元素的总和 `sum(nums[0...i])`，无论内部如何传递，其总和是不可能增加的（因为值只能从 `i+1` 传进来，不能从 `i` 传到 `i+1`）。

如果我们想让这个前缀 `nums[0...i]` 中的每一个元素都小于等于 `m`，那么这 `i+1` 个元素能容纳的最大总和就是 `(i + 1) * m`。由于操作无法增加前缀和，所以一个必要条件就是：**原数组的每一个前缀和都必须小于等于它所能容纳的最大总和**。
即，对于所有的 `i` from `0` to `n-1`，必须满足 `sum(nums[0...i]) <= (i + 1) * m`。

事实证明，这不仅是必要条件，也是充分条件。如果所有前缀和都满足这个条件，我们总能找到一种操作方式，将“高”处的值“填”到“低”处，使得最终所有元素都不超过 `m`。

因此，`check(m)` 的逻辑如下：
1.  遍历数组 `nums`，同时计算当前的前缀和 `prefix_sum`。
2.  对于每个位置 `i`，检查 `prefix_sum` 是否大于 `(i + 1) * m`。
3.  如果发现任何一个前缀和超出了这个上限，说明 `m` 太小了，无法容纳当前的前缀和，返回 `False`。
4.  如果遍历完整个数组都没有违反这个条件，说明 `m` 是一个可行的最大值，返回 `True`。

### Python 代码实现

```python
from typing import List

class Solution:
    def minimizeArrayValue(self, nums: List[int]) -> int:
        
        # 确定二分查找的左右边界
        # 答案的下界可以是 0
        left = 0
        # 答案的上界不会超过数组中的最大值
        right = max(nums)
        
        # 辅助函数：检查是否可能使数组中的最大值不超过 m
        def check(m: int) -> bool:
            # prefix_sum 可能会很大，需要使用足够大的类型
            prefix_sum = 0
            for i, num in enumerate(nums):
                prefix_sum += num
                # 检查前 i+1 个元素的总和是否超过了
                # 它们在理想情况下（所有元素都为 m）所能达到的总和
                if prefix_sum > (i + 1) * m:
                    return False
            return True

        # 二分查找寻找最小的可行 m
        while left < right:
            mid = left + (right - left) // 2
            if check(mid):
                # 如果 mid 可行，说明它可能就是答案，
                # 或者还有更小的可行值，所以我们尝试在左半部分继续寻找
                right = mid
            else:
                # 如果 mid 不可行，说明目标值太小，
                # 必须增大，所以去右半部分寻找
                left = mid + 1
        
        # 循环结束时，left 就是满足条件的最小整数
        return left
```

### 复杂度分析

*   **时间复杂度**: O(N * log(M))。
    *   `N` 是数组 `nums` 的长度。
    *   `M` 是数组中的最大值，代表了二分查找的搜索空间大小。
    *   二分查找本身需要 `log(M)` 次迭代。
    *   在每次迭代中，我们都需要调用 `check` 函数，该函数需要遍历一次 `nums` 数组，时间复杂度为 O(N)。
    *   因此，总时间复杂度是这两者的乘积。
*   **空间复杂度**: O(1)。
    *   除了输入数组外，我们只使用了几个变量来存储左右边界和前缀和，空间复杂度是常数级的。

### 总结

本题是“最小化最大值”问题类型的经典范例。解决这类问题的通用策略是使用二分查找。我们将一个难以直接求解的“最优化问题”，转化为一个相对简单的“判定性问题”。只要这个判定问题具有单调性，就可以通过二分法高效地逼近并找到最终的答案。此题的精髓在于，通过分析操作的单向性，将判定条件与前缀和巧妙地联系起来。
