LeetCode 第 2064 题：[分配给商店的最多商品的最小值](https://leetcode.cn/problems/minimized-maximum-of-products-distributed-to-any-store/description/)。
本题是一道典型的“最小化最大值”问题。其核心在于识别出答案的单调性，并将一个求解最优值的问题，巧妙地转化为一个判定性的问题，从而利用二分查找在对数时间内高效求解。
{% endcenterquote %}
<!--more-->

### 问题解读

我们需要将 `m` 种商品分配给 `n` 家商店。每种商品的数量在 `quantities` 数组中给出。分配时必须遵守以下规则：
1.  一间商店最多只能分配到**一种**商品。
2.  商店可以接收任意数量的同种商品（可以为0）。
3.  我们的目标是找到一种分配方案，使得所有商店中，商品数量最多的那一家，其商品数量（我们称之为 `x`）尽可能小。返回这个最小的 `x`。

这个问题本质上是要求我们找到一个最优的 `x`，即“最小化的最大值”。

举个例子，`n = 6`, `quantities = [11, 6]`。
*   如果我们尝试让 `x=2`（即每家商店最多拿2件商品），会发生什么？
    *   11 件商品需要 `ceil(11/2) = 6` 间商店。
    *   6 件商品需要 `ceil(6/2) = 3` 间商店。
    *   总共需要 `6 + 3 = 9` 间商店，但这超出了我们拥有的 `n=6` 间商店，所以 `x=2` 是不可行的。
*   如果我们尝试让 `x=3` 呢？
    *   11 件商品需要 `ceil(11/3) = 4` 间商店。
    *   6 件商品需要 `ceil(6/3) = 2` 间商店。
    *   总共需要 `4 + 2 = 6` 间商店，这恰好等于 `n=6`。所以 `x=3` 是一个可行的分配方案。

通过这个例子，我们可以感觉到 `x` 和所需商店数量之间的一种反比关系：`x` 越大，分配同一种商品所需的商店就越少，总商店数也可能越少。我们的任务就是在所有可行的 `x` 中，找到那个最小的值。

### 核心思路：将“求解问题”转化为“判定问题”与二分查找

直接求解“最小的可能 `x`”似乎比较复杂。但我们可以换一个角度思考：如果我们**假设**一个 `x` 的值，我们能否判断出这个值是否可行？

这个问题就简单多了。我们可以写一个辅助函数 `check(mx)`，来判定当每家商店最多分配 `mx` 件商品时，能否在 `n` 家商店内完成所有分配。
*   对于 `quantities` 数组中的每一种商品 `q`，为了保证每家商店不超过 `mx` 件，我们需要 `ceil(q / mx)` 间商店。在整数运算中，这等价于 `(q + mx - 1) // mx`。
*   我们将所有商品类型所需要的商店数加起来，得到一个总和 `total_stores`。
*   最后，比较 `total_stores` 和 `n`。如果 `total_stores <= n`，说明 `mx` 是一个可行的最大值。

现在，我们拥有了一个判定方法，并且可以发现这个判定结果具有**单调性**：
*   如果一个值 `mx` 是可行的（即所需商店数 `≤ n`），那么任何大于 `mx` 的值 `mx'` 也必然可行（因为它需要的商店数只会更少或相等）。
*   反之，如果 `mx` 不可行，那么任何小于 `mx` 的值也必然不可行。

这种单调性是应用二分查找的完美信号。我们可以对 `x` 的可能取值范围进行二分查找，来快速定位到那个“最小可行”的临界点。

这个二分查找的搜索范围是什么呢？
*   **下界 (left)**：`x` 至少为1才有意义。在代码实现中，我们可以从0开始作为搜索下界。
*   **上界 (right)**：在最极端的情况下，一种数量最多的商品可能全部分配给一家商店，所以 `x` 的上界不会超过 `max(quantities)`。

因此，我们就在 `[1, max(quantities)]` 这个区间内，对 `x` 进行二分查找。

### 算法详解

1.  **确定二分查找的范围**
    *   `l = 0`：一个不可能的下界，用于二分查找的边界。
    *   `r = max(quantities)`：一个肯定可行的上界。

2.  **执行二分查找**
    *   使用 `while l + 1 < r` 的循环条件，这是一种在整数域上查找下界的常用模板。
    *   计算中间值 `mid = (l + r) // 2` 作为当前的“猜测最大值”。
    *   调用辅助函数 `check(mid)` 来判定这个猜测值是否可行。
        *   **如果 `check(mid)` 返回 `True`** (即 `mid` 这个最大值是可行的)：
            *   说明 `mid` 是一个潜在的答案，但我们想找的是“最小”的 `x`，所以我们尝试一个更小的值。我们将 `mid` 作为新的上界：`r = mid`。
        *   **如果 `check(mid)` 返回 `False`** (即 `mid` 这个最大值太小，导致所需商店超过 `n`)：
            *   说明我们需要允许商店分配更多的商品，所以 `mid` 不可能是答案。我们将 `mid` 作为新的下界：`l = mid`。

3.  **返回结果**
    *   循环结束时，`l` 和 `r` 相邻（`l + 1 == r`）。根据我们的更新规则，`l` 是最后一个不满足条件的值，而 `r` 是第一个满足条件的值。因此，`r` 就是我们要求的最小的可能的 `x`。

### Python 代码实现

```python
from typing import List

class Solution:
    def minimizedMaximum(self, n: int, quantities: List[int]) -> int:
        """
        使用二分查找来寻找最小化的最大值x。
        """
        
        # 辅助函数：检查在每家商店最多分配 mx 件商品的情况下，
        # 所需的总商店数是否不超过 n。
        def check(mx: int) -> bool:
            # mx为0会导致除零错误，在我们的二分范围l=0, r>0时，mid永远不会是0。
            if mx == 0:
                return False
            cnt = 0
            for x in quantities:
                # 计算分配数量为x的商品需要多少家商店
                # (x + mx - 1) // mx 是向上取整的整数运算技巧
                cnt += (x + mx - 1) // mx
            return cnt <= n

        # 二分查找的范围
        # l 是一个不可能的下界，r 是一个可能的上界
        l, r = 0, max(quantities)
        
        # 当 l 和 r 之间至少还有一个整数时，继续循环
        while l + 1 < r:
            mid = (l + r) // 2
            if check(mid):
                # 如果 mid 可行，说明它可能就是答案，
                # 或者还有更小的可行值，所以我们将上界缩小到 mid
                r = mid
            else:
                # 如果 mid 不可行，说明最大值太小了，
                # 必须增大，所以我们将下界提升到 mid
                l = mid
        
        # 循环结束时，r 是第一个满足 check(r) == True 的值
        return r
```

### 复杂度分析

*   **时间复杂度**: O(M * log(K))。
    *   `M` 是商品种类的数量，即 `len(quantities)`。
    *   `K` 是商品数量的最大值，即 `max(quantities)`。二分查找的搜索空间大小为 `K`。
    *   二分查找本身需要 `log(K)` 次迭代。
    *   在每次迭代中，我们都需要调用 `check` 函数，该函数需要遍历一次 `quantities` 数组，时间复杂度为 O(M)。
    *   因此，总时间复杂度是这两者的乘积。
*   **空间复杂度**: O(1)。
    *   除了输入数据外，我们只使用了少数几个变量来辅助计算，空间复杂度是常数级的。
