LeetCode 第 1648 题：[销售价值减少的颜色球](https://leetcode.cn/problems/sell-diminishing-valued-colored-balls/description/)。
本题的核心在于如何高效处理大规模订单下的贪心策略。通过观察答案的单调性，我们可以将一个求解最大利润的问题，转化为一个判定性的问题（某个价值阈值是否可行），从而利用二分查找在对数时间内锁定最优解的范围。
{% endcenterquote %}
<!--more-->

### 问题解读

我们有一批库存，包含多种颜色的球。顾客要购买 `orders` 个球。这道题的特别之处在于球的价值计算方式：一个球的价值等于**卖出它之前**，库存中同色球的数量。这意味着，如果我们有 10 个红球，卖掉第一个红球价值 10，第二个价值 9，以此类推。

我们的目标是制定一个销售策略，来最大化卖出 `orders` 个球的总价值。

这个问题的贪心思路非常直观：**在任何时候，我们都应该选择当前价值最高的球来卖**。价值最高的球，就是来自当前库存数量最多的那种颜色的球。

例如，库存是 `[2, 5]`，要卖 4 个。
1.  当前库存最多的是 5 个球的那种，价值为 5。卖一个，库存变为 `[2, 4]`，总价值 5。
2.  现在库存最多的是 4 个球的那种，价值为 4。卖一个，库存变为 `[2, 3]`，总价值 5 + 4 = 9。
3.  现在库存最多的是 3 个球的那种，价值为 3。卖一个，库存变为 `[2, 2]`，总价值 9 + 3 = 12。
4.  现在两种球都剩 2 个，价值都是 2。随便卖一个，库存变为 `[1, 2]`，总价值 12 + 2 = 14。

这个过程如果用一个最大堆（Priority Queue）来模拟，每次取出最大值，减一后再放回去，重复 `orders` 次。但题目的数据范围 `orders` 可以达到 10^9，O(orders * logN) 的复杂度显然会超时。我们需要更高效的方法。

### 核心思路：将“贪心求解”转化为“二分答案”

既然逐个销售太慢，我们可以考虑“成批”销售。观察贪心过程，我们总是在卖光所有价值为 `v` 的球之后，才开始卖价值为 `v-1` 的球。这启发我们，一定存在一个“价值阈值” `k`，我们的销售策略是：
1.  把所有价值 **大于** `k` 的球全部卖掉。
2.  再卖掉一部分价值 **等于** `k` 的球，直到满足 `orders` 的数量要求。

这个价值阈值 `k` 是解题的关键。如果我们能快速找到它，问题就迎刃而解。

这个阈值 `k` 具有单调性：
*   我们设定的阈值 `k` 越高，意味着我们只卖价值非常高的球，卖出的球的总数就越少。
*   反之，阈值 `k` 越低，卖出的球的总数就越多。

卖出球的数量是关于阈值 `k` 的单调递减函数。这种单调性正是二分查找的绝佳应用场景。我们可以对这个“价值阈值 `k`”进行二分查找。

我们的目标是找到一个临界值 `k`，使得所有价值 > `k` 的球加起来的数量，刚好小于 `orders`，而所有价值 > `k-1` 的球加起来的数量，则会大于等于 `orders`。

这个 `k` 就是我们最后卖出的那批球的价值。

### 算法详解

1.  **确定二分查找的范围**
    *   `left = 0`：价值阈值的下限。
    *   `right = max(inventory)`：价值阈值的上限，不可能超过初始最多的球数。

2.  **定义判定函数 `count_balls(k)`**
    *   这个函数用于计算：如果我们将阈值设为 `k`，即所有价值严格大于 `k` 的球都卖掉，总共能卖出多少个球。
    *   实现很简单，遍历 `inventory`，对于每个库存 `inv`，如果 `inv > k`，则可以卖出 `inv - k` 个球。将这些数量求和即可。
    *   `count_balls(k) = sum(max(0, inv - k) for inv in inventory)`

3.  **执行二分查找**
    *   当 `left <= right` 时，循环继续。
    *   计算中间值 `mid = left + (right - left) // 2` 作为当前的“猜测阈值”。
    *   调用 `count_balls(mid)` 来判定：
        *   **如果 `count_balls(mid) >= orders`**：说明把所有价值 > `mid` 的球都卖掉，数量已经足够或超额了。这意味着我们最终的阈值 `k` 应该**不低于** `mid`。所以，我们尝试一个更高的阈值，`left = mid + 1`。
        *   **如果 `count_balls(mid) < orders`**：说明即使把所有价值 > `mid` 的球都卖掉，数量仍然不够 `orders`。这意味着我们的阈值设得太高了，必须降低才能卖出更多的球。所以，`right = mid - 1`。
    *   循环结束后，`right` (或者 `left-1`) 就是我们苦苦寻找的那个临界阈值 `k`。

4.  **计算总价值**
    *   找到阈值 `k` 后，总利润的计算分为两部分：
    *   **第一部分：价值 > k 的球**
        *   遍历 `inventory`，对于每个 `inv > k` 的库存，我们卖掉了从 `inv` 到 `k+1` 的所有球。这是一个等差数列求和。
        *   卖出的数量是 `count = inv - k`。
        *   价值总和是 `(inv + k + 1) * count / 2`。
        *   我们将这部分的利润和卖出的球数累加起来。
    *   **第二部分：价值 = k 的球**
        *   用总订单数 `orders` 减去第一部分卖出的球数，得到还需要卖多少个球，我们称之为 `remaining_orders`。
        *   这 `remaining_orders` 个球的价值都是 `k`。
        *   所以这部分的利润是 `remaining_orders * k`。
    *   将两部分利润相加，就是最终答案。注意，在计算过程中要时刻对 `10^9 + 7` 取余，防止整数溢出。

### Python 代码实现
```python
from typing import List

class Solution:
    def maxProfit(self, inventory: List[int], orders: int) -> int:
        MOD = 10**9 + 7

        # 辅助函数：计算如果所有价值 > k 的球都卖掉，能卖出多少个
        def count_balls(k: int) -> int:
            count = 0
            for inv in inventory:
                if inv > k:
                    count += (inv - k)
            return count

        # 二分查找寻找价值阈值 k
        # k 是我们最终售出的球中，价值最低的那个球的价值
        left, right = 0, max(inventory)
        k = 0
        while left <= right:
            mid = (left + right) // 2
            if mid == 0: # 避免 mid-1 出现负数
                left = mid + 1
                continue
            
            # 如果把价值 > mid-1 的球都卖了，数量还不够 orders
            # 说明阈值太高了，需要降低
            if count_balls(mid - 1) < orders:
                right = mid - 1
            else:
                # 否则，mid 可能就是我们的阈值 k，或者 k 更大
                # 我们希望 k 尽可能大，以获得更高利润
                k = mid
                left = mid + 1
        
        # 计算总利润
        total_profit = 0
        orders_sold = 0

        # 第一部分：计算所有价值 > k 的球带来的利润
        for inv in inventory:
            if inv > k:
                # 卖出从 inv 到 k+1 的所有球
                num_balls = inv - k
                orders_sold += num_balls
                # 等差数列求和公式: (首项 + 末项) * 项数 / 2
                profit = (inv + k + 1) * num_balls // 2
                total_profit = (total_profit + profit) % MOD
        
        # 第二部分：卖出剩余的球，它们的价值都是 k
        remaining_orders = orders - orders_sold
        total_profit = (total_profit + remaining_orders * k) % MOD
        
        return total_profit

```

### 复杂度分析

*   **时间复杂度**: O(N * log(M))。
    *   `N` 是库存数组 `inventory` 的长度。
    *   `M` 是 `inventory` 中的最大值，即二分查找的搜索空间大小。
    *   二分查找本身需要 `log(M)` 次迭代。
    *   在每次迭代中，我们都需要调用 `count_balls` 函数，该函数需要遍历一次 `inventory` 数组，时间复杂度为 O(N)。
    *   因此，总时间复杂度是这两者的乘积。
*   **空间复杂度**: O(1)。
    *   除了输入数组外，我们只使用了几个变量来存储左右边界、中间值和计算过程中的状态，所以空间复杂度是常数级的。

### 总结

本题是“二分答案”思想的经典应用。当直接求解一个最优值（如本题的最大利润）很困难，但验证一个给定的答案（“如果最终卖出的最低价值是k，能否满足条件？”）相对容易时，就可以考虑使用二分查找。如果验证函数具有单调性，我们就能将求解问题转化为判定问题，通过不断缩小答案的搜索范围，在对数级别的时间复杂度内找到最优解，这对于处理大规模数据至关重要。
