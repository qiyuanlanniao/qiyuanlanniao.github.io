LeetCode 1898 | 可移除字符的最大数目

发表于 2025-11-02 分类于 算法题解 ， 二分算法 ， 二分答案 ， 求最大 阅读次数： Valine：

LeetCode 第 1898 题：可移除字符的最大数目。
本题旨在求解一个“最大值”，其核心性质具有单调性。通过巧妙地将这个“求解最优值”的问题转化为一个“判定可行性”的问题，我们可以利用二分查找在对数时间内高效地锁定最终答案。

问题解读

我们得到两个字符串 s 和 p（p 是 s 的一个子序列），以及一个包含 s 中字符下标的数组 removable。题目要求我们从 removable 数组中按顺序取出前 k 个下标，并将 s 中对应位置的字符移除。我们的目标是找到一个最大的 k，使得在移除了这 k 个字符之后，p 仍然是 s 的子序列。

这里的核心约束条件是：

  1. 移除操作必须严格按照 removable 数组给出的顺序进行，即只能选择 removable 的一个前缀 [0, ..., k-1]。
  2. 移除字符后，s 中剩余字符的相对顺序保持不变。
  3. 我们需要找到使 p 依然是 s 的子序列的那个最大的 k。

举个例子，s = "abcacb", p = "ab", removable = [3,1,0]。

  • 如果我们尝试 k=1，移除 removable[0] 即 s[3]，s 变为 "abccb"。"ab" 是其子序列。可行。
  • 如果我们尝试 k=2，移除 removable[0] 和 removable[1] 即 s[3] 和 s[1]，s 变为 "accb"。"ab" 仍然是其子序列。可行。
  • 如果我们尝试 k=3，移除 removable[0], removable[1], removable[2] 即 s[3], s[1], s[0]，s 变为 "ccb"。"ab" 不再是其子序列。不可行。

因此，满足条件的最大 k 就是 2。

核心思路：将“求解问题”转化为“判定问题”与二分查找

直接去寻找这个最大的 k 可能比较棘手。但是，我们可以换一个角度思考：如果我们假定一个 k 值，我们能否判断出这个 k 是否满足条件？

这个判定问题相对简单。我们可以写一个辅助函数 is_subsequence(k)，它的功能是：检查当移除 removable 数组前 k 个下标对应的字符后，p 是否还是 s 的子序列。
这个函数可以这样实现：

  1. 将 removable 数组的前 k 个下标存入一个哈希集合（Set），以便后续能以 O(1) 的时间复杂度快速判断某个下标是否需要被移除。
  2. 使用双指针法来检查子序列关系。一个指针 p_ptr 指向 p 的开头，另一个指针 s_ptr 指向 s 的开头。
  3. 遍历字符串 s（用 s_ptr）。如果 s_ptr 指向的下标不在移除集合中，并且 s[s_ptr] 等于 p[p_ptr]，那么我们就成功匹配了一个字符，将 p_ptr 向后移动一位。
  4. 遍历结束后，如果 p_ptr 已经走完了整个 p 字符串（即 p_ptr == len(p)），那就说明 p 是修改后 s 的一个子序列，k 是可行的。

现在，我们有了这个判定的方法。我们可以观察到这个判定结果具有非常重要的单调性：

  • 如果移除 k 个字符后 p 是 s 的子序列，那么移除更少的字符（例如 k-1 个），s 中留下的字符会更多，p 自然也是 s 的子序列。
  • 反之，如果移除 k 个字符后 p 不是 s 的子序列，那么移除更多的字符（例如 k+1 个），s 中留下的字符会更少，p 就更不可能成为 s 的子序列了。

这种 [是, 是, ..., 是, 否, 否, ...] 的性质，正是应用二分查找的完美场景。我们可以对 k 的取值范围进行二分查找，来快速定位那个从“是”到“否”的临界点，也就是我们想要的最大的可行 k。

这个二分查找的搜索范围是什么呢？

  • 下界 (left)：可以一个都不移除，所以 k 的最小值是 0。
  • 上界 (right)：最多可以把 removable 数组里所有的下标都移除，所以 k 的最大值是 len(removable)。

因此，我们就在 [0, len(removable)] 这个区间内，对 k 进行二分查找。

算法详解

  1. 确定二分查找的范围

    • left = 0：可能的最小 k 值。
    • right = len(removable)：可能的最大 k 值。
    • ans = 0 初始化结果，用来记录我们找到的最大的可行 k。

  2. 执行二分查找

    • 当 left <= right 时，循环继续。
    • 计算中间值 mid = left + (right - left) // 2 作为当前的“猜测 k 值”。
    • 调用辅助函数 is_subsequence(mid) 来判定这个猜测值是否可行。
      • 如果 is_subsequence(mid) 返回 True (即移除 mid 个字符是可行的)：
        • 说明 mid 是一个潜在的答案。但我们想找的是“最大”的 k，所以我们尝试一个更大的值。我们先将当前的可行解记录下来 ans = mid，然后将搜索范围缩小到右半边：left = mid + 1。
      • 如果 is_subsequence(mid) 返回 False (即移除 mid 个字符后 p 不再是子序列)：
        • 说明 mid 这个 k 值太大了，我们需要减少移除的字符数量。将搜索范围缩小到左半边：right = mid - 1。

  3. 返回结果

    • 循环结束后，ans 中存储的就是满足条件的最大 k 值。

复杂度分析

  • 时间复杂度: O(log(R) * (M + R))。
    • M 是字符串 s 的长度，R 是 removable 数组的长度。
    • 二分查找的搜索空间大小是 R+1，因此二分本身需要 log(R) 次迭代。
    • 在每次迭代中，我们都需要调用 is_subsequence 函数。在该函数内部：
      • 构建哈希集合需要 O(k) 的时间，k 是当前的 mid，最大为 R。
      • 双指针检查子序列需要遍历一次 s，时间复杂度为 O(M)。
    • 因此，is_subsequence 函数的复杂度为 O(M + k)。
    • 总时间复杂度是 log(R) * O(M + R)。鉴于题目限制 R < M，也可记为 O(M * log(R))。
  • 空间复杂度: O(R)。
    • 在 is_subsequence 函数中，我们创建了一个哈希集合 removed_indices 来存储要移除的下标。在最坏的情况下，其大小可以达到 R。因此，空间复杂度为 O(R)。

总结

本题是“二分答案”思想的典型应用。当我们面对求解满足特定条件的“最大值”或“最小值”问题时，如果该条件具有单调性，就可以优先考虑是否能将其转化为一个判定问题。通过将复杂的求解问题简化为重复的、更简单的判定问题，并结合二分查找，我们能将时间复杂度从线性级别（或更高）优化到对数级别，从而高效地解决问题。
