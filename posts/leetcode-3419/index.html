LeetCode 3419 | 图的最大边权的最小值

发表于 2025-11-15 分类于 算法题解 ， 二分算法 ， 二分答案 ， 最小化最大值 阅读次数： Valine：

LeetCode 第 3419 题：图的最大边权的最小值。
当直接求解一个最优值非常困难时，我们可以尝试猜测一个答案，然后去验证这个猜测是否可行。这种将“求解问题”转化为“判定问题”的思路，是解决这类问题的关键。

问题解读

我们需要在一个有向带权图中删除一些边，目标是找到一个边的子集，同时满足三个条件：

  1. 连通性: 从任何一个其他节点出发，都必须存在一条路径能够到达节点 0。
  2. 出度限制: 在我们选择的边的子集中，每个节点的出度（即从该节点出发的边的数量）不能超过 threshold。
  3. 最小化最大边权: 在满足以上两个条件的前提下，我们选择的边的子集中，权重最大的那条边的权重（最大边权）应该尽可能小。

我们需要返回这个“最小的可能的最大边权”。如果无法满足条件，则返回 -1。

核心思路：将“求解问题”转化为“判定问题”与二分查找

这道题要求我们“最小化最大边权”，这是一个非常强烈的信号，提示我们可以使用二分查找来解决。我们不对边的集合进行搜索，而是对**答案本身（即最大边权）**进行二分查找。

假设我们猜测一个最终答案为 max_w。那么问题就从“找到最小的那个值”变成了“是否存在一个方案，使得最大边权不超过 max_w？”。这个“是否存在”的问题就是一个判定问题，通常比原问题更容易解决。

我们可以设计一个函数 check(max_w)，它的功能是：

  • 接收一个 max_w 作为参数。
  • 判断我们是否可以只使用权重 w <= max_w 的边，从中挑选出一个子集，满足“所有节点可达节点 0”和“出度不超过 threshold”这两个条件。

这个 check(max_w) 函数具有非常明显的单调性：

  • 如果一个较小的最大边权 max_w 是可行的，那么任何一个更大的 max_w' (因为它允许我们使用更多的边) 也一定是可行的。
  • 反之，如果 max_w 不可行，那么任何更小的 max_w'' (因为它可用的边更少) 也必然不可行。

这种单调性确保了我们可以通过二分查找，高效地找到那个“刚刚好”满足条件的最小 max_w。

check(max_w) 的实现逻辑

现在，核心问题变成了如何实现 check(max_w)。
给定一个 max_w，我们首先可以筛选出所有权重 w <= max_w 的边，构成一个子图。

  1. 处理连通性条件: “所有节点都能到达节点 0” 这个条件，对于每个节点都要检查一遍，似乎有点麻烦。这里有一个常用的技巧：反向建图。如果我们把子图中所有边的方向都反转过来，那么原来的条件就等价于：“在反向图中，从节点 0 出发，可以到达所有其他节点”。这个问题就简单多了，我们只需要从节点 0 开始进行一次深度优先搜索（DFS）或广度优先搜索（BFS），然后看是否能访问到全部 n 个节点即可。

  2. 处理出度限制: “每个节点的出度不超过 threshold“。这个条件怎么办？我们是不是要在检查连通性的时候，小心翼翼地选择边，以确保出度不超标？
    其实，这个条件可以被大大简化。要满足“所有节点可达节点 0”，我们实际上只需要一个边的子集，这个子集构成一个以 0 为根的“有向生成树”（或称为“树状子图”，Arborescence）。在这个结构中，除了根节点 0，每个其他节点都有且仅有一条出边，指向它在路径上的下一个节点。
    因为题目保证了 threshold >= 1，所以一个出度为 1 的方案是完全满足出度限制的。
    这意味着：只要我们能用权重 <= max_w 的边让图连通（即所有节点可达0），我们就一定能从中找出一个满足出度限制的子集。
    因此，在 check(max_w) 函数中，我们只需要关心连通性问题，可以暂时忽略出度限制！

算法详解

  1. 预处理: 如果总边数 len(edges) 小于 n-1，那么无论如何都不可能将 n 个节点连通起来，直接返回 -1。
  2. 确定二分范围:
    • left = 0：最大边权的最小值可能是 0。
    • right = 10^6 (或 max(w for _,_,w in edges))：最大边权的最大可能值。
  3. 执行二分查找:
    • 当 left <= right 时循环。
    • 计算中间值 mid = left + (right - left) // 2，作为当前猜测的“最大边权”。
    • 调用 check(mid)：
      • 在 check(mid) 内部，构建一个反向图，但只加入那些原始权重 w <= mid 的边。
      • 从节点 0 开始进行 DFS 或 BFS，统计可访问到的节点数量。
      • 如果访问到了 n 个节点，说明 mid 这个最大边权是可行的，check(mid) 返回 True。否则返回 False。
    • 更新边界:
      • 如果 check(mid) 返回 True：说明 mid 是一个潜在的答案，但我们想找更小的，所以记录下来 ans = mid，然后尝试在左半部分 [left, mid-1] 继续寻找。
      • 如果 check(mid) 返回 False：说明 mid 太小了，必须允许更大的边权，所以去右半部分 [mid+1, right] 寻找。
  4. 返回结果: 循环结束后，ans 中存储的就是最小的可行最大边权。如果一次都没有成功过，则返回 -1。

复杂度分析

  • 时间复杂度: O(E * log(W_max))，其中 E 是边的数量，W_max 是最大的边权。
    • 二分查找的搜索空间是 [0, W_max]，因此迭代次数为 O(log(W_max))。
    • 在每次迭代中，我们都需要调用 check 函数。check 函数内部需要遍历所有 E 条边来建图 (O(E))，然后进行一次图的遍历（BFS/DFS），其复杂度为 O(N + E’)，其中 E’ 是权重小于等于 mid 的边数，E' <= E。
    • 因此，总时间复杂度是 O((N+E) * log(W_max))。由于 E 通常大于 N，可以简化为 O(E * log(W_max))。
  • 空间复杂度: O(N + E)。
    • 空间开销主要来自于 check 函数中存储反向图的邻接表 g，最坏情况下需要存储所有边，即 O(N+E)。
    • BFS/DFS 中的队列和 visited 集合也需要 O(N) 的空间。

总结

通过将求解“最小的最大值”问题转化为“判定一个给定的最大值是否可行”的问题，我们成功地利用了答案的单调性，并通过二分查找高效地锁定了最优解。其中，反向建图来简化连通性判断，以及分析并简化threshold约束是解决本题的两个关键技巧。当遇到最优化问题时，不妨思考一下是否能将其转化为判定问题，这或许能为你打开一扇新的大门。
