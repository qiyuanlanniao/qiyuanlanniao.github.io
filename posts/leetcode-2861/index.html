LeetCode 2861 | 最大合金数

发表于 2025-11-05 分类于 算法题解 ， 二分算法 ， 二分答案 ， 求最大 阅读次数： Valine：

LeetCode 第 2861 题：最大合金数。
本题的核心思路是，对于每一台机器，可以制造的合金数量都具有单调性。利用这一特性，我们可以将求解“最大值”的问题转化为一个“判定性”问题，并通过二分查找高效地找到答案。

问题解读

我们扮演一个合金制造公司老板的角色，目标是在给定的预算（budget）内，最大化合金的生产数量。

核心的约束条件和信息如下：

  1. 有 k 台不同的机器，但我们一次只能选择一台机器进行生产。
  2. 每台机器制造一个合金需要不同配方（composition）。
  3. 我们有一定数量的初始金属库存（stock）。
  4. 如果库存不足，我们可以按给定的价格（cost）购买任意数量的金属。
  5. 最终的目标是，在所有机器中，找到一个能使生产数量最大化的方案，并返回这个最大数量。

举个例子，假设用 1 号机器生产 10 个合金的总成本是 100，用 2 号机器生产 12 个合金的总成本是 120。如果我们的预算是 110，那么我们选择 1 号机器，最终答案是 10。如果预算是 130，我们选择 2 号机器，答案就是 12。

我们的任务就是在所有可行的方案中，找到那个最大的合金生产数。

核心思路：将“求解问题”转化为“判定问题”与二分查找

直接求解“最大生产数”比较复杂，因为它同时涉及到选择哪台机器和确定生产多少数量。我们可以简化问题：先不考虑哪台机器最好，而是针对某一台特定的机器，计算它在预算内最多能生产多少合金。如果我们能解决这个子问题，那么最终答案就是所有 k 台机器对应的最大生产数中的最大值。

现在，我们聚焦于单台机器。如何找到它的最大生产数？
我们可以发现一个关键的单调性：如果这台机器在预算内可以生产 x 个合金，那么它一定也可以生产 x-1 个合金（因为成本更低）。反之，如果预算不足以生产 x 个合金，那么也肯定无法生产 x+1 个合金（因为成本更高）。

这种单调性是应用二分查找的绝佳信号。我们可以对“合金生产数量”这个答案进行二分查找，来快速定位那个“预算内可行的最大值”。

为了实现二分查找，我们需要一个判定函数 check(num)，它的功能是：对于当前这台机器，生产 num 个合金是否可行（即总成本是否会超过预算 budget）？

check(num) 函数的逻辑如下：

  • 初始化总花费 money = 0。
  • 遍历生产 num 个合金所需要的每一种金属：
    • 计算需要第 j 种金属的总量：required = composition[i][j] * num。
    • 查看库存 stock[j]，如果库存不足（stock[j] < required），则计算需要购买的数量 required - stock[j]。
    • 将购买所需的花费 (required - stock[j]) * cost[j] 累加到 money 中。
    • 如果在任何一步，money 已经超过了 budget，就可以提前判定为不可行，返回 False。
  • 如果遍历完所有金属，money 仍然没有超过 budget，则说明 num 是一个可行的生产数量，返回 True。

有了这个判定函数，我们就可以为每一台机器进行二分查找了。搜索范围是什么呢？

  • 下界 (left)：最少可以生产 0 个合金。
  • 上界 (right)：一个宽松但安全的上界是 min(stock) + budget。因为制造一个合金最少也需要花费 1 块钱（如果成本最低为1且无库存），因此用尽预算最多也就能买 budget 份材料。加上初始最少的库存，这是一个非常安全的上限。

算法详解

  1. 初始化

    • ans = 0，用来记录在所有 k 台机器中能找到的全局最大合金数。

  2. 遍历每台机器

    • 对 k 台机器的配方 composition 进行遍历。对于每台机器，我们都执行一次二分查找来计算它能生产的最大合金数。

  3. 对单台机器执行二分查找

    • 确定范围：
      • 左边界 l = ans (一个优化，因为我们不关心比已找到的答案更小的数)。
      • 右边界 r = min(stock) + budget + 1 (一个安全的上界)。
    • 循环查找：
      • 当 l 和 r 没有相邻时 (l + 1 < r)，循环继续。
      • 计算中间值 mid = (l + r) // 2 作为当前的“猜测生产数”。
      • 调用 check(mid) 函数来判定这个猜测值是否可行。
        • 如果 check(mid) 返回 True：
          • 说明生产 mid 个合金是可行的，那么我们应该尝试生产更多。所以 mid 是一个潜在的解，我们将搜索范围的下界更新为 mid，即 l = mid。
        • 如果 check(mid) 返回 False：
          • 说明生产 mid 个合金成本太高，我们必须减少生产数。将搜索范围的上界更新为 mid，即 r = mid。
    • 更新答案：
      • 当二分查找循环结束时，l 就代表了当前这台机器在预算内能生产的最大合金数。
      • 用 l 来更新全局最大值 ans。

  4. 返回结果

    • 遍历完所有 k 台机器后，ans 中存储的就是最终的答案。

Python 代码实现

复杂度分析

  • 时间复杂度: O(k * n * log(M))
    • k 是机器的数量，我们有一个外层循环遍历所有机器。
    • 对于每台机器，我们执行一次二分查找。搜索空间的大小 M 大约为 budget + min(stock)。因此二分查找的迭代次数是 log(M)。
    • 在二分查找的每一次迭代中，我们都会调用 check 函数，该函数需要遍历 n 种金属，时间复杂度为 O(n)。
    • 所以，总时间复杂度是这三者的乘积。
  • 空间复杂度: O(1)
    • 除了输入数据外，我们只使用了几个变量来存储边界和计算成本，没有使用与输入规模成正比的额外空间。

总结

本题是“二分答案”思想的典型应用。当我们求解一个“最大值”或“最小值”问题，并且发现答案的范围具有单调性时，就可以考虑将问题转化为一个判定问题（“某个值是否可行？”）。然后利用二分查找，将问题的规模不断减半，从而在对数时间内高效地锁定最优解。对于本题，我们对每一台机器都应用了这个策略，最终取所有结果中的最大值。
