LeetCode 2517 | 礼盒的最大甜蜜度

LeetCode 第 2517 题：礼盒的最大甜蜜度。
这是一个经典的“最大化最小值”问题。当直接求解“最大的甜蜜度”比较困难时，我们可以转换思路，通过二分查找来猜测一个甜蜜度 d，然后验证是否存在一种方案能够满足“任意两种糖果的价格差都大于等于 d”，从而将一个求解最值的问题转化为一个判定性的问题。

题目描述

给你一个正整数数组 price ，其中 price[i] 表示第 i 类糖果的价格，另给你一个正整数 k 。

商店组合 k 类 不同 糖果打包成礼盒出售。礼盒的 甜蜜度 是礼盒中任意两种糖果 价格 绝对差的最小值。

返回礼盒的 最大 甜蜜度。

解题思路

问题转换与二分答案

题目的目标是求“最大”的“最小”差值，这是一个典型的“最大化最小值”问题，通常可以考虑使用二分查找来解决。

我们不直接去求解这个最大甜蜜度，而是去猜测它。假设我们猜测最大甜蜜度可以是 d，那么我们需要解决一个判定性问题：能否在 price 数组中选出 k 个数，使得它们之间任意两个数的差都大于等于 d ？

这个判定问题具有单调性：

  • 如果甜蜜度 d 可以实现，那么任何小于 d 的甜蜜度（例如 d-1）也一定可以实现。
  • 如果甜蜜度 d 无法实现，那么任何大于 d 的甜蜜度（例如 d+1）也一定无法实现。

这种单调性是使用二分查找的理论基础。我们可以二分枚举答案 d，找到满足条件的 d 的最大值。

check(d) 函数的设计

现在，核心问题变成了如何高效地实现 check(d) 函数，即判断能否选出 k 个价格，使得它们两两之差的绝对值不小于 d。

为了让价格差尽可能大，我们应该选择那些数值上离得尽可能远的糖果。一个自然的想法是贪心。

  1. 排序：首先，我们将 price 数组升序排序。这样，任意两个价格的差值就等于它们在排序后数组中的数值之差。
  2. 贪心选择：我们总是选择第一个糖果（价格最低的那个）作为礼盒中的第一个成员。然后，向后遍历数组，寻找下一个与上一个已选糖果价格之差大于等于 d 的糖果。
    • 设我们上一个选择的糖果价格是 pre。我们从 pre 的位置继续向后扫描，找到第一个价格 p 满足 p - pre >= d。
    • 一旦找到，就将这个糖果选入礼盒，更新 pre = p，并将计数器加一。
    • 重复这个过程，直到遍历完整个数组。
  3. 判断：最后，检查我们通过这种贪心策略选出的糖果数量是否达到了 k。如果数量大于等于 k，说明甜蜜度 d 是可行的，check(d) 返回 True；否则返回 False。

这个贪心策略是正确的，因为为了在有限的价格范围内放下尽可能多的、满足最小间距 d 的糖果，我们每次选择下一个糖果时，都应该选择“恰好”满足条件或者刚刚超过条件的那个，这样可以为后面的选择留出最大的空间。

二分查找框架

  1. 确定搜索范围：
    • 甜蜜度的最小值显然是 0。
    • 甜蜜度的最大值不会超过 (price[n-1] - price[0])。我们可以用这个作为上界，或者一个更紧的上界 (price[n-1] - price[0]) / (k - 1)。
  2. 实施二分：
    • 设置左边界 l = 0，右边界 r 为一个足够大的数（例如 10^9 或上述上界）。
    • 在 while 循环中，取中点 mid = (l + r) // 2。
    • 调用 check(mid)：
      • 如果 check(mid) 为 True，说明 mid 这个甜蜜度是可行的，我们可以尝试更大的值。所以，我们将 mid 存为可能的答案，并移动左边界 l = mid + 1。
      • 如果 check(mid) 为 False，说明 mid 太大了，需要尝试小一点的值。移动右边界 r = mid - 1。
  3. 返回结果：循环结束后，最后一次使得 check 返回 True 的那个值就是最终答案。

代码实现

复杂度分析

  • 时间复杂度：O(N log N + N log M)，其中 N 是 price 数组的长度，M 是价格的最大差值 price[-1] - price[0]。
    • 对 price 数组排序需要 O(N log N) 的时间。
    • 二分查找的过程需要进行 O(log M) 次迭代。
    • 在每次迭代中，check 函数需要 O(N) 的时间来遍历数组。
    • 因此，总时间复杂度为 O(N log N + N log M)。由于 N 通常远大于 log M，复杂度主要由排序和 check 函数决定。
  • 空间复杂度：O(log N) 或 O(N)，取决于排序算法所使用的栈空间。如果原地排序，则为 O(log N)；如果使用了需要额外空间的排序，则为 O(N)。
